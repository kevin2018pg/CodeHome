1. 先建立定位

MMoE / PLE 主要解决什么

Task-centric

核心问题是多个任务之间怎么共享、怎么隔离、怎么减少负迁移。

典型表达：shared experts + task gates + task towers。

RankMixer 主要解决什么

Feature-token-centric

核心问题是多域稀疏特征怎么高效交互，排序模型怎么扩容，同时线上延迟可控。

典型表达：tokens + token mixing + per-token FFN。

一句话记忆

Backbone

RankMixer 更像工业排序 backbone，不是 MMoE/PLE 的简单替代品。

它可以接 CTR/CVR 多任务 head，也可以和 MMoE/PLE 结合。

面试表达：MMoE/PLE 关注任务共享，RankMixer 关注特征交互、模型扩容和硬件效率。两者不是同一个层面的设计。

3. Block 内部结构

Token Mixing 的直觉

它让不同语义 token 的不同 channel 发生交互，但不计算 attention map。

你可以把它理解成“硬件友好的跨域通道重排”。

Per-token FFN 的直觉

user/item/context/sequence/cross 的分布不同，所以每个 token 用自己的 FFN 更合理。

这保留了特征域差异，同时仍然能统一堆叠模型。

5. 离线训练链路

训练侧必须关心

  • 按 request_id 保留候选列表，用 NDCG/GAUC 评估。
  • CTR/CVR 标签窗口要固定，CVR 要考虑延迟回传。
  • 训练和推理使用同一份特征配置、hash、缺失值和归一化。
  • CVR 稀疏时要考虑 loss 权重、负采样、样本校准。

当前仓库对应

feature_config.json 定义 token 和处理方式。

train.py 训练 CTR/CVR 双任务。

infer.py 输出 ctr_pred、cvr_pred、rank_mixer_score。

6. 线上推理链路

延迟

关注 P50/P90/P99，而不是只看平均耗时。候选数、token 数、embedding 维度、MoE top-k 都会影响延迟。

特征

线上必须有缺失默认值和类型校验。ID 类特征要保证 hash 和离线完全一致。

监控

看 CTR/CVR 分布、校准、特征缺失率、expert 激活分布、GPU 利用率和线上 A/B。

7. 面试高频题卡

Q1：RankMixer 解决了什么问题？

解决工业排序模型扩容时的效率问题。它用 token mixing 做多域特征交互，用 per-token FFN 保留特征域差异，并通过 Sparse-MoE 扩大容量、控制推理成本。

Q2：RankMixer 和 Transformer 有什么区别？

Transformer 用 self-attention，表达强但成本高；RankMixer 不生成 attention map，而是通过 channel/token 维度混合做交互，更硬件友好。

Q3：为什么需要 Per-token FFN？

因为 user、item、context、sequence、cross 的语义和分布不同。Per-token FFN 保留各自子空间，避免所有 token 共用同一变换导致表达被抹平。

Q4：RankMixer 和 MMoE/PLE 怎么结合？

RankMixer 可以作为 backbone 输出 shared representation，然后接 MMoE/PLE 或多个 task head。RankMixer 解决特征交互，MMoE/PLE 解决任务共享。

Q5：Sparse-MoE 的线上风险是什么？

专家负载不均、路由不稳定、跨设备通信、部分专家训练不足、fallback 难。需要监控 expert 激活率、路由熵、P99 延迟和线上指标。

Q6：item_id/user_id 是字符串还是数字？

原始可以是字符串或 int64，但模型里必须按 categorical ID 处理，通常 hash/vocab 后进 embedding，不能当连续数值。

Q7：离线 AUC 涨了线上一定涨吗？

不一定。还要看校准、NDCG、分场景指标、延迟、线上候选集、position bias、特征新鲜度和业务融合公式。

8. 一周复习路径

Day 1-2：会讲结构

背熟：feature grouping -> tokenization -> token mixing -> per-token FFN -> CTR/CVR heads。

Day 3：会对比

重点对比 MMoE、PLE、Transformer、DeepFM/DCN。每个说清解决的问题和代价。

Day 4-5：会落地

掌握特征 schema、训练标签、评估指标、推理延迟、训推一致、线上监控。

Day 6-7：会面试

拿上面的题卡练习，不只答结构，还要答为什么适合工业场景。

最终口径：RankMixer 是面向工业排序扩容的 backbone。它以特征 token 为中心，通过高效 token mixing 和 per-token FFN 建模多域特征交互，并用 Sparse-MoE 在参数规模和线上成本之间做平衡。
