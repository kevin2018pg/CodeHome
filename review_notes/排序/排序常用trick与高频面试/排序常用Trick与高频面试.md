# 排序(精排) 常用 Trick 与高频面试题 · 实战手记

> 视角：排序算法工程师。这篇是**精排侧的 trick 速查 + 面试串讲**：模型演进、特征交叉、行为序列、多目标、校准、偏差、评估，一条线串清。和召回篇 [`召回常用Trick与高频面试`](../../召回/召回常用trick与高频面试/召回常用Trick与高频面试.md) 对照看。

---

## 一、心法：精排在链路里干啥

- **位置**：召回(亿级→几千) → 粗排(几千→几百) → **精排(几百→几十排准)** → 重排(打散/多样性/业务规则)。
- **目标**：在召回/粗排送来的几百个候选里，**精准预估每个 item 的 pCTR/pCVR/时长等**，按业务目标排序。
- **和召回最大的不同**：
  - 样本 = **曝光样本**(正=点击/转化，负=曝光未点击)，训练空间 ≈ 预测空间，不需要像召回那样补全库负样本。
  - 结构**可以任意特征交叉**(不用解耦双塔)，user×item×context 怎么交叉都行 → 模型重心就在"怎么把交叉和序列建好"。
  - 在意**绝对值校准**(pCTR 要准，下游要用它算 ecpm/出价)，不只是相对序。

> 一句话：召回求"全不全"，精排求"准不准 + 校得准"。

---

## 二、模型演进主线（面试必背）

| 阶段 | 代表 | 解决了什么 | 遗留问题 |
|---|---|---|---|
| 线性 | **LR + 大规模特征工程** | 简单可解释、能上规模 | 全靠人工组合特征，不会自动交叉 |
| 自动二阶交叉 | **FM / FFM** | 自动学二阶交叉、解决稀疏 ID 共现少 | 只到二阶 |
| 记忆+泛化 | **Wide & Deep** | wide 记忆 + deep 泛化 | wide 侧仍需人工交叉 |
| 自动+深度 | **DeepFM / DCN / xDeepFM** | FM/Cross 自动交叉 + DNN 高阶 | 交叉是隐式/bit 级，可解释弱 |
| 行为序列 | **DIN / DIEN / DSIN** | 用 target 对历史行为做 attention，刻画兴趣 | 长序列算不动 |
| 超长序列 | **SIM / ETA / MIMN** | 检索式处理万级长期行为 | 工程复杂 |
| 多任务 | **ESMM / MMoE / PLE** | 多目标联合、缓解 CVR 样本偏差/跷跷板 | 任务关系、融合权重要调 |

要点：**DeepFM = FM(显式二阶) + DNN(隐式高阶) 共享 embedding；DCN = Cross 网络做有界高阶显式交叉。DIN 的灵魂是 target attention(不同候选下用户兴趣表示不同)。**

## 三、特征工程 Trick（精排真正的大头）

- **ID 类 + embedding**：高维稀疏 ID(user/item/类目/作者) 走 embedding，是主力。
- **交叉特征**：user×item、user×类目、context×item 等人工交叉仍很有用(给模型先验)。
- **统计/后验特征**：item 历史 CTR、user 类目偏好率、滑窗统计——**注意防穿越**(只能用 t 时刻之前的统计)。
- **连续特征处理**：分桶(等频/等宽)、log 变换、归一化；分桶比直接喂原值更稳。
- **特征频次过滤 & 低频 ID 处理**：低频 ID 合并到默认/hash 桶，防过拟合。
- **特征重要性 & 上线纪律**：离线/线上特征对齐(同一套抽取逻辑)，防训练-serving skew。

> 易错：**特征穿越(label leakage)**——用了包含未来信息的统计特征，离线 AUC 虚高、线上崩。

---

## 四、特征交叉 Trick

- **FM**：`<v_i, v_j>` 自动学二阶，解决稀疏共现。
- **DCN / DCN-v2**：Cross 层 `x_{l+1} = x_0 * (w·x_l) + b + x_l`，显式有界高阶交叉，参数少。
- **DeepFM**：FM 与 DNN **共享 embedding**，显式二阶 + 隐式高阶并联。
- **xDeepFM(CIN)**：vector-wise 显式高阶交叉(比 DCN 的 bit-wise 更可解释)。
- **AutoInt**：multi-head self-attention 做特征交叉。
> 选型：工业基线常 **DeepFM / DCN-v2**，够用且稳；要序列就上 DIN 系。

---

## 五、用户行为序列建模 Trick

- **DIN(target attention)**：用候选 item 当 query，对用户历史行为算 attention 权重 → **同一用户面对不同候选，兴趣表示不同**。这是精排序列建模的分水岭。
- **DIEN**：加 GRU + 兴趣演化层(AUGRU)，建模兴趣随时间漂移。
- **SIM / ETA**：长期行为(万级)先检索(GSU 硬搜同类目/软搜向量 top-k)再 attention(ESU)，把长序列压成相关短序列。和召回里序列长度的处理同源。
- **多行为序列**：点击/加购/购买分开建模再融合。

> 追问常考：**DIN 的 attention 和普通 attention 区别？** → DIN 用 target 当 query 且**不做 softmax 归一化**(保留兴趣强度的绝对量级)，激活用 Dice/PReLU。

---

## 六、多目标 / 多任务 Trick（高频考点）

- **为什么多目标**：业务要同时优化 CTR、CVR、时长、完播…单目标顾此失彼。
- **ESMM**：在**全曝光空间**建模 `pCTR` 和 `pCTCVR`，`pCVR = pCTCVR / pCTR`，解决 **CVR 的样本选择偏差(只在点击样本上训会偏)** 和样本稀疏。
- **MMoE**：多个专家 + 每个任务一个 gating，软共享，缓解任务冲突。
- **PLE / CGC**：把专家分成**任务专属 + 共享**，分层抽取，进一步缓解 MMoE 的"跷跷板/负迁移"。
- **多目标融合**：线上把多个预估按公式融合排序，如

```
score = pCTR^α * pCVR^β * (期望时长)^γ * ...   # 乘法
或  score = w1*pCTR + w2*pCVR + ...            # 加法(需各项校准/量纲一致)
```

权重 α/β/γ 用线上 A/B 或多目标帕累托调，是业务和算法的交界。

> 追问：**MMoE 和 PLE 区别？** → PLE 显式区分共享/专属专家并分层，专门治 MMoE 的负迁移(任务相关性低时 MMoE 退化)。

---

## 七、校准(Calibration) Trick

- **为什么要校准**：广告/竞价里 pCTR 要算 `ecpm = bid × pCTR`，**绝对值必须准**，不能只对相对序。
- **导致不准的原因**：负采样(改变了正负比)、样本不均衡、多目标融合。
- **做法**：
  - **负采样后校准**：`p_calibrated = p / (p + (1-p)/w)`，w 是负采样率，纠回真实先验。
  - **Platt scaling / 等频分桶保序回归(Isotonic)**：在验证集上拟合一层校准。
  - **PCOC(Predicted CTR / Observed CTR)** 当监控指标，趋近 1 才算校准好。

---

## 八、偏差(Bias) Trick

- **位置偏差(position bias)**：高位天然点击多，模型会把"高位"学成"好"。
  - **PAL / 位置特征只在训练用**：训练时把 position 当独立特征(或独立塔)，**线上推断置默认位**，剥离位置影响。
- **选择偏差/幸存者偏差**：只在曝光样本上训，没曝光的没标签 → 和召回 SSB 同源，精排相对轻但 CVR 任务明显(ESMM 治这个)。
- **流行度偏差**：热门 item 后验 CTR 高，统计特征会放大，注意 debias / 时间衰减。

---

## 九、训练 / 工程 Trick

- **在线学习 / 增量更新**：FTRL(稀疏)、增量训练，跟上分布漂移、保时效。
- **embedding 维度与冲突**：大规模 ID 用 hash embedding / 动态 embedding，控内存。
- **训练-serving 一致性**：特征抽取同一套逻辑、同样的分桶边界、同样的默认值，防 skew(精排头号工程坑)。
- **样本加权**：按业务价值(成交额)、时间衰减给样本加权。
- **蒸馏**：大模型/多目标 teacher 蒸小模型，平衡效果与时延。

---

## 十、评估 Trick

- **AUC**：整体排序能力，但**跨用户**比较，会被"用户间差异"稀释。
- **GAUC(Group AUC)**：按 user 分组算 AUC 再按曝光/点击加权平均，**更贴近线上单用户内排序**，精排主看 GAUC。
- **校准指标**：PCOC、LogLoss、校准曲线。
- **多目标**：各任务 AUC/GAUC + 线上大盘(CTR、CVR、时长、GMV)，看融合后整体。
- **线上**：A/B 看核心指标 + 护栏指标(时长、留存、多样性)。

> 高频追问：**AUC 涨了线上不涨？** → ① AUC 是跨用户、线上是用户内(看 GAUC)；② 校准没做好(序对值不对)；③ 离线特征穿越；④ 多目标融合权重没调；⑤ 训练-serving skew。

## 十一、高频面试题（带追问）

**Q1：精排和召回/粗排的区别？**
位置(几百→几十)、样本(曝光样本，空间≈预测)、结构(可任意交叉)、目标(要校准绝对值)。

**Q2：DeepFM 为什么比 Wide&Deep 好？FM 和 DNN 怎么配合？**
Wide 侧仍需人工交叉；DeepFM 用 FM 自动学二阶 + DNN 学高阶，**共享 embedding**，免人工。

**Q3：DCN 的 Cross 网络在干啥？和 DNN 区别？**
Cross 做**显式有界高阶交叉**(每层和 x0 交互)，参数少、可解释；DNN 是隐式交叉。两者并联互补。

**Q4：DIN 的核心思想？attention 和普通 attention 区别？**
target attention：候选当 query 对历史行为加权，不同候选下用户兴趣不同。区别：不做 softmax 归一化(保留强度)，用 Dice 激活。
> 追问：DIEN 在 DIN 上加了啥？ → 兴趣抽取 GRU + 兴趣演化 AUGRU，建模兴趣漂移。

**Q5：长期行为序列(万级)怎么建模？**
SIM/ETA：GSU 检索(硬搜同类目/软搜向量)出相关 top-k，ESU 再 attention。和召回序列长度处理同源。

**Q6：ESMM 解决什么问题？怎么解决的？**
CVR 的**样本选择偏差**(只在点击样本训会偏)和稀疏。在全曝光空间建 pCTR、pCTCVR，`pCVR=pCTCVR/pCTR`。

**Q7：MMoE 和 PLE 区别？什么时候用 PLE？**
MMoE 软共享专家 + 任务 gating；任务相关性低时会负迁移/跷跷板。PLE 分共享/专属专家并分层，缓解负迁移。

**Q8：多目标怎么融合成一个排序分？**
乘法 `pCTR^α·pCVR^β·时长^γ` 或加法加权，权重靠 A/B / 帕累托调；加法要先校准量纲。

**Q9：为什么要校准？pCTR 不准会怎样？**
竞价 `ecpm=bid×pCTR` 依赖绝对值；负采样/不均衡会让 p 偏。用负采样校准公式 / Isotonic，监控 PCOC。

**Q10：位置偏差怎么处理？**
训练时 position 当特征(或 PAL 独立塔)，线上置默认位，剥离位置影响。

**Q11：AUC 和 GAUC 区别？精排看哪个？**
AUC 跨用户、被用户间差异稀释；GAUC 用户内分组加权，更贴线上。精排主看 GAUC。

**Q12：离线 AUC 涨线上不涨，怎么排查？**
GAUC vs AUC、校准、特征穿越、融合权重、训练-serving skew。

**Q13：连续特征怎么处理？为什么分桶？**
分桶(等频)+ embedding 比直接喂原值更稳、抗异常值、能学非线性；也可 log/归一化。

**Q14：怎么防特征穿越？**
统计/后验特征只用 t 时刻前的数据；按时间切分训练/验证；上线前查时间字段。

**Q15：精排负样本要不要像召回那样补全库？**
不用。精排训练空间≈预测空间(都是曝光)，用曝光未点击当负即可；补全库反而引入分布偏差。

## 十二、易错 / 陷阱题

- **"精排也要补未曝光负样本"** ❌ → 那是召回逻辑；精排曝光样本即可。
- **"AUC 高就一定线上好"** ❌ → 看 GAUC + 校准，AUC 跨用户会失真。
- **"pCTR 只要相对序对就行"** ❌ → 竞价/出价要绝对值，必须校准。
- **"统计特征越多越好"** ❌ → 易穿越、易过拟合，要防 leakage + 时间衰减。
- **"位置特征线上也照常喂"** ❌ → 会把高位偏差带进预估，线上要置默认位。
- **"多目标 MMoE 一定比单目标好"** ❌ → 任务相关性低会负迁移，可能要 PLE 或拆模型。
- **"DIN 的 attention 要 softmax 归一化"** ❌ → DIN 故意不归一化以保留兴趣强度。

---

## 十三、一句话总结

> 精排 trick 四条线：**特征(交叉+统计+防穿越)**、**结构(FM/DCN 交叉 + DIN/SIM 序列)**、**多目标(ESMM 治偏差 + MMoE/PLE 治冲突 + 融合权重)**、**可信(校准 + 偏差剥离 + GAUC 评估)**。面试主线：**精排是"在曝光空间里把 pCTR/pCVR 预估得又准又校得对"，所以交叉、序列、多任务、校准、偏差，是它最常被深挖的地方。**
