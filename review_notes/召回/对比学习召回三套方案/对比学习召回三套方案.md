# 对比学习语义召回 · 三套方案（搜索语义相似 / I2I / U2I）· 实战手记

> 视角：从 NLP 语义相似(SimCSE)迁移到推荐召回。三套方案共享同一套"骨架"——**bi-encoder 双塔 + 对比学习 + L2norm/温度 + ANN 检索**，差别只在**样本怎么构建**和**塔怎么搭**。面试常问"你 NLP 的经验怎么用到推荐"，这篇就是答案。
>

## 〇、三套方案共享的"骨架"（先记这个，后面只讲差异）

| 共享组件 | 内容 |
|---|---|
| 架构 | **bi-encoder 双塔**：两侧各自独立出向量(可分解 `f(a)·g(b)`)，才能离线建索引 |
| 损失 | **对比学习**：in-batch negatives + InfoNCE/CrossEntropy(`labels=arange`，对角线为正) |
| 度量 | **L2 normalize + 温度 τ**(0.05 量级)，内积=余弦 |
| 负样本 | in-batch 负 + 难负 + 跨 GPU 负(all_gather 扩规模) |
| 索引 | 向量灌 Milvus/Faiss，`metric=IP`(已归一化即 cosine)，IVF/HNSW |
| serving | 编码服务(TorchServe)产 query/item 向量 → ANN top-K |

> **核心红线**：必须 bi-encoder（双塔各自独立编码），不能 cross-encoder（两个输入拼一起算分）——后者不可分解、无法建索引，只能做精排/重排。

三套方案的**唯一本质差异 = 正样本对怎么定义**：

```
方案一(搜索)：正样本 = (query, doc) 语义相关对      → 学“意思像不像”
方案二(I2I) ：正样本 = (item, item) 行为/内容相关对 → 学“物品相不相关”
方案三(U2I) ：正样本 = (user, item) 真实交互对      → 学“用户会不会喜欢”
```

## 一、先讲清 SimCSE 是什么（基础）

SimCSE = Simple Contrastive Learning of Sentence Embeddings，是把句子编码成向量的对比学习方法。核心就一句：**拉近正样本对、推开负样本**。分两种：

### 1.1 无监督 SimCSE（最经典的 trick）
- **正样本对怎么来**：**同一句话，过两次 encoder，用不同的 dropout mask**，得到两个略有差异的向量——这两个就是正样本对。dropout 本身当成"数据增强"。
- **负样本**：同 batch 里其它句子。
- 学到的是**通用文本语义表示**（"这句话 ≈ 它自己的另一种 dropout 视图"）。
- 局限：只学"文本自身像不像"，学不到任务相关性。

### 1.2 有监督 SimCSE（效果更好）
- 用带标注的句对（如 NLI）：`(前提, 蕴含句)` = 正样本对，`(前提, 矛盾句)` = **hard 负样本**。
- 训练成三元组 `(anchor, positive, hard_negative)`，hard 负显式加进对比。矛盾句是真正意义上的 hard negative——语法相近、语义相反，让模型学到更细粒度的语义边界。
- 你那套 STS 项目就是**有监督版**：GPT 造的 `(query, 资源描述)` 当正样本对 + 难负，学的是"搜索相关性"，比无监督强很多。

**为什么 SimCSE 不只是"又一个 InfoNCE 方法"（高频追问）：**

SimCSE 之前，BERT 句向量有个公认的问题：**各向异性（Anisotropy）**——所有向量挤在高维空间的一个狭窄锥形区域里，随机两句话的余弦相似度都高达 0.9+，完全区分不开：
```
BERT 原始向量分布：      对比学习后：
  ↑↑↑↑↑↑↑↑↑           ↑  →  ↙  ↗
   ↑↑↑↑↑↑↑           ←   ↘    ↑  →
    ↑↑↑↑↑↑          ↓      ↗  ←  ↙
  所有向量挤一块        均匀散开在超球面上
```

SimCSE 是**第一个把这个病根诊断清楚并量化的**：用 Alignment + Uniformity 两个指标说明，BERT 的问题不是 Alignment 差（正样本不够近），而是 **Uniformity 极差**（所有向量挤一堆，没区分度）。InfoNCE 里"推开负样本"这件事，本质上就是在强制优化 Uniformity。

另一个贡献是发现：**dropout 是文本最好的数据增强**。之前大家用删词、换词、裁剪，结果一般，因为文本增强容易改变语义。dropout 只加轻微扰动、语义完全保留，是理想的"同语义不同视角"。这两个洞察让 SimCSE 成了那几年语义相似的标配。

### 1.3 损失（InfoNCE，三套方案通用）

InfoNCE 原始公式（对第 i 个 anchor）：
```
        exp(sim(a_i, p_i) / τ)
L_i = -log ─────────────────────────────────
            Σ_j exp(sim(a_i, b_j) / τ)     # j 遍历 batch 内所有 b（含正样本）
```

**InfoNCE = CrossEntropy(B×B相似度矩阵, arange(B))，两者完全等价：**

batch 内有 B 个正样本对，编码后得 A、B 两个矩阵，算出 B×B 相似度矩阵 S：
```
S = A · Bᵀ / τ

        b_0    b_1    b_2
a_0  [  s00    s01    s02 ]   ← 第0行：a_0 对所有 b 的相似度
a_1  [  s10    s11    s12 ]   ← 对角线 s_ii = sim(a_i, b_i)，即正样本对
a_2  [  s20    s21    s22 ]   ← 非对角线是 in-batch 负样本
```

- **对角线 = 正样本**：s00 = sim(a_0, b_0)，它们是配对的。
- **同行其余 = 负样本**：s01 = sim(a_0, b_1)，a_0 和 b_1 没有配对关系。
- **每行做 softmax** = "从 B 个 b 里选出 a_i 的正样本"，等价于 B 分类问题，label = 对角线位置 i。

所以 `CrossEntropy(S, arange(B))` 就是对所有行的 InfoNCE 求平均。**PyTorch 没有官方 InfoNCELoss，但它就等于这三行**，不需要额外的库：
```python
S      = (A @ B.T) / tau        # B×B 相似度矩阵
labels = torch.arange(B)        # [0, 1, 2, ..., B-1]
loss   = F.cross_entropy(S, labels)
```

`sim` 用 L2norm 后的内积（= 余弦），τ 温度控制分布尖锐程度（τ 越小越尖锐，对难负越敏感，推荐 0.05 量级）。

### 1.4 为什么有效：Alignment & Uniformity（高频追问）
对比学习同时优化两件事：
- **Alignment（对齐）**：正样本对要靠近——拉近正样本对做的事；
- **Uniformity（均匀）**：所有向量在单位超球面上**尽量均匀散开**——推开负样本做的事。

**BERT 的核心病是 Uniformity 差，不是 Alignment 差。** BERT 预训练没有显式推开负样本的机制，导致所有句向量挤在一个锥形区域（各向异性），Uniformity 极差。InfoNCE 里"推开 in-batch 负样本"直接修复了这一点，让向量均匀散开在超球面上，余弦相似度才有了真正的区分度。

Uniformity 好的向量**天然适合 ANN 检索**：向量均匀分布意味着 ANN 索引能切出更有意义的分区，检索精度更高。这也是对比学习产出的向量比普通分类/回归向量更好用的根本原因。

### 1.5 SimCSE 和双塔召回的关系（关键认知）
> **SimCSE = 共享 encoder 的对比学习；双塔召回 = 两个独立塔的对比学习。训练框架同源。**

你 SimCSE 代码里的核心几行，就是双塔召回的标准训练写法：
```python
z1 = F.normalize(z1)                             # [B, dim]  a 向量(query/user)
z2 = F.normalize(z2)                             # [B, dim]  b 向量(doc/item)
# unsqueeze(1): [B,1,dim]，unsqueeze(0): [1,B,dim]
# 广播后逐元素乘再 sum → [B,B] 相似度矩阵，等价于 z1 @ z2.T
cos_sim = sim(z1.unsqueeze(1), z2.unsqueeze(0))  # [B, B] / τ
labels  = arange(B)                              # [0,1,...,B-1]，对角线是正样本
loss    = CrossEntropy(cos_sim, labels)          # InfoNCE
```
迁到推荐，只需把"共享 BERT 编码 query/doc"换成"**两个独立塔编码 user/item**"，正样本对换成行为对，**这段 loss 代码原样复用**。

## 二、对比学习训练的核心流程（三套通用）

### 2.1 先分清：双塔的三种训法，只有第三种是对比学习

| 训法 | 怎么训 | 是不是对比学习 |
|---|---|---|
| pointwise(老 DSSM) | 每条 (a,b) 单独打分，正 label=1 负=0，**BCE/sigmoid** | ❌ 普通二分类 |
| pairwise | BPR：正样本分 > 负样本分 | 半对比 |
| **listwise / in-batch softmax** | batch 内 1 正 + 多负做 softmax | ✅ **这就是对比学习** |

所以"引入对比学习" = 把 loss 从 **逐条 BCE** 换成 **batch 内 in-batch 负 + softmax**。结构不变，只改样本组织和 loss。

### 2.2 核心流程 5 步（背这个）

1. **组 batch 的正样本对**：`(a_1,b_1),…,(a_B,b_B)`，每条都是真实相关/交互对，**不用显式造负**。
2. **两塔各自编码 + L2 normalize**：得到 `A=[a_1..a_B]`、`B=[b_1..b_B]`。
3. **算 B×B 相似度矩阵**：`S = A·Bᵀ / τ`。**对角线是正样本，同行其余 B−1 个是 in-batch 负**。
4. **InfoNCE**：每行 softmax，`labels=arange(B)`，CrossEntropy。
5. **(推荐场景再加)**：logQ 校正(治热门) + 全局随机负(治 SSB) + 中段难负(提精度)。搜索/纯语义可不加。

### 2.3 为什么用对比(in-batch softmax)而不是 pointwise BCE
- **负样本利用率高**：B 个正样本自动提供 B×(B−1) 个负样本，不用单独采；
- **直接优化检索目标**："从一批里挑对的"，比二分类更贴召回；
- **向量质量好**：uniformity 让分布均匀，利于 ANN。

## 三、方案一：Item/Query 语义相似（搜索场景，我的原方案）

### 3.1 定位
旅游域搜索召回：用户自然语言 query → 检索相关资源(度假/酒店/景点)。本质 **query→doc 语义匹配**，BERT 直接可用。

### 3.2 样本怎么构建（关键）
- **正样本对 = (query, 资源描述)**，三个来源拼起来：
  1. **GPT 造对**：ChatGPT 给每个资源 title 生成对应用户 query（解决标注稀缺）；
  2. **有监督相似对**：已有的人工/业务相似样本；
  3. **BART 无监督改写增强**：对 query/描述做同义改写扩量。
- **负样本**：① in-batch 内其它 doc；② 随机远距离配对造负；③ 难负(三元组里的 z3)。
- 监督信号 = **语义相关**（不是纯 dropout，所以是"有监督 SimCSE"，比无监督强）。

> 记忆点：**SimCSE 学到"哪种相似"完全由正样本对决定**。dropout → 文本自身；GPT query-doc 对 → 搜索相关性。

### 3.3 网络框架
```
query / doc
   → BERT/TCBert(先在旅游语料做领域 MLM 继续预训练)
   → pooling(avg_top2，最后两层平均)
   → MLP 投影(Linear→128 + Tanh)
   → L2 normalize
   → 128 维向量
相似度 = cos(z_query, z_doc) / τ(0.05)，in-batch CrossEntropy
```
- **共享 encoder**：query、doc 用同一个 BERT。
- 输出 128 维。

### 3.4 核心优化点
- **领域 MLM 继续预训练**：通用 BERT 先适配旅游域再对比微调。
- **GPT 造正样本对**：低成本拿到大量 query-doc 相关对。
- **难负 + 跨 GPU 负**(all_gather)：扩负样本规模、提边界。
- **FGM 对抗训练**：扰动 word embedding，提鲁棒/泛化。
- **拼音增强**：文本向量 + 拼音向量融合，抗错别字/同音。
- **avg_top2 pooling**：比单 [CLS] 更稳；**L2norm + τ=0.05** 配 IP 索引。

### 3.5 线上
TorchServe(`text2vec`) 编码 query → Milvus(`IVF_FLAT`, `IP`, dim=128, nlist=1024, nprobe=32) top-10。资源向量离线抽好入库。

### 3.6 一句话
**领域 MLM + GPT 造对 + 有监督 SimCSE(难负/对抗/拼音)→ 128 维向量 → Milvus 检索**。它也是下面 I2I/U2I 的"内容编码器"基础。

## 四、方案二：I2I（物品-物品召回）

### 4.1 定位
给定用户近期交互的 item(trigger)，召回相关 item。**迁移成本最低**——方案一的"query↔doc"换成"item↔item"几乎零改造。两条腿：**内容语义 I2I** 和 **行为 I2I**。

### 4.2 样本怎么构建（决定学"内容相似"还是"行为相似"）

| 正样本对来源 | 怎么造 | 学到的相似 | 用途 |
|---|---|---|---|
| 内容相似对 | 同 item 不同字段/改写，或同类内容对 | **内容/语义相似** | 冷启动、长尾 |
| **共现/共购对** | 同一用户**先后/同 session 交互**的 (itemA, itemB) | **行为相似**(最有用) | 主力召回 |
| 弱监督对 | 同类目/同品牌/同属性 | 弱语义相似 | 补充 |

- **负样本**：in-batch 负 + 随机全库负(治 SSB) + 中段难负(避 false negative，剔历史正反馈)。
- 关键：**要"行为相似"就必须用共现/session 正样本对**；纯内容对只给"长得像"，抓不到手机↔壳膜这类互补。

### 4.3 网络框架（两种）
**(a) 纯内容 bi-encoder（直接复用方案一）**
```
itemA文本 ┐
          ├ 共享 BERT 编码器(可复用方案一权重) → pooling → MLP → L2norm → 向量
itemB文本 ┘
正样本对 = (共现 itemA 文本, 共现 itemB 文本)，in-batch 对比
```

**(b) ID+内容混合 item 塔（更推荐）**
```
item → [ item ID embedding ⊕ 类目/属性 embedding ⊕ BERT内容向量 ] → MLP → L2norm → 向量
两侧共享同一 item 塔；正样本 = 共现 item 对
```
- BERT 内容向量当一个特征(可冻结/微调)，ID embedding 补行为协同。

### 4.4 核心优化点
- **正样本对设计**(内容对 vs 共现对)——决定召回性质，最重要。
- **共现(Swing) + 语义向量双路融合**：Swing 准、抓行为、压热门；语义向量泛化、抓冷启/长尾。工业标配。
- **难负 + false negative 过滤**；L2norm + τ；ANN(HNSW/IVF)。
- 内容向量天然解决**新品冷启**(发布即编码就有向量)。

### 4.5 线上
trigger item → 查其向量 → ANN top-N → 多 trigger 加权融合(近因/行为权重) → 去重过滤。

### 4.6 一句话
**I2I = 方案一双塔搬到 item↔item，正样本换共现/内容对；语义向量(BERT) + 共现(Swing)双路融合**，内容路解决冷启、共现路保准确。

## 五、方案三：U2I（用户-物品召回，最“推荐”的形态）

### 5.1 定位
直接学"用户↔物品"匹配：user 塔出用户向量、item 塔出物品向量，ANN 召回。改造最大——**user 侧不是文本，是行为序列+画像**，基模型从"文本 BERT"换成"推荐双塔"。

### 5.2 样本怎么构建
- **正样本 = 用户真实交互对 (user, item)**：CTR 用点击、CVR/GMV 用转化；多目标可多种正样本加权。
- **负样本(召回命门)**：
  - **主体 = 未曝光随机全库负**(治 SSB，让模型见全库)；
  - in-batch 负(高效但偏热门 → logQ 校正)；
  - **MNS 混合负采样**(in-batch + 全局均匀补长尾)；
  - 少量**中段难负** + false negative 过滤；
  - **不要**只用曝光未点击(那是精排逻辑)。
- 详见 [`负样本挖掘和召回效果评估`](../负样本挖掘和召回效果评估/负样本挖掘和召回效果评估.md)。

### 5.3 网络框架（两个独立塔）
```
用户塔: [ user ID ⊕ 画像/上下文 ⊕ 行为序列编码(SASRec 式 self-attention) ] → MLP → L2norm → u
物品塔: [ item ID ⊕ 类目/属性 ⊕ BERT内容向量(可选) ]                        → MLP → L2norm → v
score = u · v / τ ；in-batch + 全局负 ；InfoNCE(+logQ)
```
- **行为序列用 self-attention(SASRec)**，**不能用 target attention(DIN)**——否则 user 向量依赖候选 item，破坏可索引性。
- BERT 在这里是 **item 塔的内容编码器**(产语义向量当特征)，不是主干。
- 多兴趣需求 → user 塔出 K 个向量(MIND/ComiRec)。

### 5.4 核心优化点
- **负采样体系**：未曝光随机负 + MNS + logQ + 难负(U2I 效果一大半在这)。
- **序列建模**：行为序列 self-attention 抓即时意图。
- **L2norm + 温度**(可学习)；**内容向量融合**缓解新品冷启。
- **信息瓶颈缓解**：SENet / 多向量 / 精排蒸召回。
- **训练-serving 一致**：度量、归一化、特征抽取对齐；user 向量与 item 索引同模型版本。

### 5.5 线上
线上算 1 个 user 向量 → ANN 检索全库 item top-K。item 向量离线全量入库，user 向量实时/近线算。

### 5.6 一句话
**U2I = 推荐双塔(user 序列塔 / item ID+内容塔) + 对比学习 + 负采样体系(随机负+MNS+logQ)**；BERT 退居 item 内容编码器；序列必须 self-attention 保可索引。

## 六、三套方案对比（背这张表）

| 维度 | 方案一·搜索语义 | 方案二·I2I | 方案三·U2I |
|---|---|---|---|
| 学什么 | query-doc 语义相关 | item-item 相关 | user-item 匹配 |
| 正样本怎么构建 | GPT 造 query-doc 对 | 共现/内容 item 对 | 用户真实交互 (u,i) |
| 负样本 | in-batch+随机+难负 | in-batch+随机全库+难负 | 未曝光随机+MNS+logQ+难负 |
| 基模型 | BERT(直接用，共享) | BERT 或 ID+内容塔 | 推荐双塔(序列塔/ID塔，独立) |
| BERT 角色 | 主干编码器 | 主干 或 内容特征 | item 内容编码器(配角) |
| 改造成本 | — | 低 | 高 |
| 强项 | 文本检索 | 冷启/长尾/相关 | 行为个性化 |

---

## 七、从 NLP 迁移到推荐的路线（从易到难）

1. **复用现有**：方案一的 SimCSE/BERT → 做**语义 I2I**(正样本换共现/内容对)，灌 Milvus 当冷启/长尾通路。最快见效。
2. **范式迁移**：搭 **U2I 推荐双塔**，user 塔上 SASRec 序列，沿用对比学习训练范式 + 负采样体系。
3. **融合增强**：item 塔塞入 BERT 内容向量，U2I 也获语义泛化；线上 I2I + U2I 多路融合。

> 关键认知：**训练范式(对比+双塔+ANN)直接搬；基模型看特征(纯内容用 BERT、推荐特征换双塔、最实用是 BERT 当内容编码器);监督信号从"语义对"换成"行为对"——这是迁移的灵魂。**

## 八、高频面试题

**Q1：SimCSE 是什么？无监督版的正样本怎么来？**
对比学习句向量。无监督版：**同一句过两次 encoder、不同 dropout** 得正样本对，batch 内其它句当负样本。有监督版用 NLI 正/难负对。核心优化 alignment(正样本近) + uniformity(向量均匀)。

**Q2：你 NLP 的语义相似经验怎么迁移到推荐？**
范式(bi-encoder+对比+L2norm/温度+ANN)直接复用；基模型按特征换；最关键是正样本对从"语义对"换成"行为对"。

**Q3：正常双塔有对比学习吗？怎么引入？**
看 loss：老式 pointwise BCE 没有；现代 in-batch softmax 就是对比。引入 = 把 loss 改成 batch 内 B×B 相似度矩阵 + arange 标签 + CrossEntropy(5 步)。

**Q4：I2I 和 U2I 哪个迁移成本低？**
I2I 低——query-doc 双塔改 item-item 几乎零改造，正样本换共现对；U2I 高——user 侧要换成行为序列建模(SASRec)。

**Q5：为什么 U2I 用户塔不能用 DIN 的 target attention？**
会让 user 向量依赖候选 item，破坏可分解性、无法 ANN。只能 self-attention。

**Q6：纯 SimCSE(dropout 正样本)做推荐够吗？**
只学文本语义、抓内容相似，抓不到行为/互补。要把正样本换成共现/交互对，或预训练+行为微调。

**Q7：SimCSE 和其他用了 InfoNCE 的方法有什么区别？**
InfoNCE loss 本身不是 SimCSE 独有的，MoCo/CLIP 都用。SimCSE 的真正贡献是两点：① **诊断了 BERT 句向量的病根**——各向异性导致向量挤一堆、余弦相似度失效，并用 Alignment + Uniformity 量化说清楚了 InfoNCE 为什么能治这个病（推开负样本 = 强制优化 Uniformity）；② **发现 dropout 是文本最好的增强**——比删词/裁剪等显式增强更好，因为语义完全保留只加轻微扰动。诊断框架干净，所以成了那几年语义相似的标配。

**Q8：为什么不直接用 InfoNCE 库，而是手写几行 CrossEntropy？**
因为 InfoNCE 就等于 `CrossEntropy(B×B相似度矩阵, arange(B))`，PyTorch 没有官方 `InfoNCELoss`，但 `F.cross_entropy` 直接就能实现，三行搞定，不需要额外的库。

## 九、一句话总结

> 一套骨架(bi-encoder 对比学习 + L2norm/温度 + ANN)，三种正样本(语义对 / 共现对 / 交互对)，三种塔(共享 BERT / 内容或 ID 塔 / 推荐双塔)。SimCSE 提供训练范式(dropout/难负正样本 + InfoNCE)，推荐侧把正样本换成行为对、再加 logQ/全库负。**先用现有 BERT 做语义 I2I 快速落地，再上 U2I 推荐双塔做行为个性化，最后融合**——这就是 NLP 语义模型迁移到推荐召回的完整路径。
