# 多模态召回 · 实战手记

> 视角：召回算法工程师。多模态召回用**图文/视频/音频**等内容信号做 embedding，核心价值是**冷启动 + 语义泛化 + 跨域**——补行为类召回(双塔/I2I)在新品/长尾上的短板。

---

## 一、一句话讲清

把 item 的**内容**（图片、标题、详情、视频帧、音频）用预训练多模态模型（CLIP / BLIP / 自研双塔）编码成 embedding，用它做 i2i 近邻，或对齐到 user 向量做 u2i。**不依赖行为共现**，所以新品一上架就能被召回。

> 一句话定位：行为召回解决"老 item 推得准"，多模态召回解决"新 item / 长尾 item 推得出去"。

---

## 二、为什么需要它（核心动机）

- **冷启动**：新品没有点击/共现，双塔 item 塔(靠 id)和 I2I(靠共现)都抓瞎，但内容是现成的。
- **语义泛化**：能召回"看起来像/讲的是一回事"但从没共现过的 item。
- **跨域/跨场景**：内容向量可跨业务复用（搜索、推荐、广告共享）。

---

## 三、技术路线

### 3.1 内容 embedding 怎么来
- **图文**：CLIP / BLIP / Chinese-CLIP 等，图像塔 + 文本塔，对比学习对齐到同一空间。
- **视频**：抽帧 + 帧聚合（mean/attention）+ 标题/ASR/OCR 文本融合。
- **多字段融合**：图 + 标题 + 类目 + 详情，拼接或 cross-attention 融合成一个 item 内容向量。

### 3.2 怎么用于召回
1. **多模态 i2i**：内容向量建 ANN，trigger item 找内容近邻（最常用、最稳）。
2. **多模态 u2i**：用户历史 item 的内容向量聚合成 user 兴趣向量，检索内容库。
3. **作为特征喂双塔**：把内容向量当 item 塔的输入特征（最常见的落地形态，不单独成召回路也行）。

### 3.3 表征对齐（关键）
预训练内容向量空间 ≠ 推荐行为空间。直接用 CLIP 向量召回，往往"长得像但用户不买"。**必须用行为数据做后训练对齐**：
- 用点击/共现 item 对做对比学习微调，把内容空间"掰"到行为空间；
- 或双塔里 item 塔输入内容向量，用行为标签端到端训练。

---

## 四、版本迭代

### v0 · 直接用预训练 CLIP 向量做 i2i
能召回但相关性一般（"视觉像但意图不符"）。

### v1 · 行为对齐微调
用点击/共现对比学习微调内容向量。效果：相关性明显提升，新品 CTR 起来。

### v2 · 多字段融合
图 + 标题 + OCR/ASR + 类目 cross-attention 融合。效果：信息更全，类目混淆下降。

### v3 · 作为双塔特征 + 独立召回路并行
内容向量既进双塔 item 塔，又单独建一路多模态 i2i。效果：冷启 + 长尾覆盖双收。

## 五、常见坑

**坑 1 · 内容像但用户不买（语义≠行为）**
原因：直接用预训练向量，没对齐行为。解：行为数据后训练对齐/微调。

**坑 2 · 模态缺失**
现象：部分 item 没图/没视频/标题空。解：模态 dropout 训练、缺失模态用均值/可学习占位、按可用模态降级。

**坑 3 · 离线 embedding 成本高、更新慢**
原因：大模型推理贵。解：批量离线抽取 + 缓存；新品优先级队列实时抽；蒸馏小模型。

**坑 4 · 跨模态尺度/分布不一致**
解：各模态向量分别归一化再融合；融合层加 LayerNorm。

**坑 5 · 召回相关但重复/同质**
解：和重排多样性配合；内容去重(近重复图文)。

## 六、常见问答（带追问）

**Q1：多模态召回解决什么双塔/I2I 解决不了的问题？**
冷启动(新品无行为)和语义泛化(召回没共现过的相似 item)。内容信号不依赖行为，新品上架即可召回。

**Q2：能直接拿 CLIP 向量召回吗？**
能跑但效果一般——CLIP 学的是"视觉-文本语义相似"，不是"推荐相关/会被点击"。
> 追问·怎么对齐？ → 用行为(点击/共现)对做对比学习微调，或把内容向量当双塔特征端到端训练，把内容空间掰到行为空间。

**Q3：多模态向量是单独成召回路，还是当特征？**
两种都常见。最稳的落地是**当 item 塔特征**(提升整体泛化)；要专门补冷启则**单独成一路多模态 i2i**。工业上常并行。

**Q4：模态缺失怎么办？**
模态 dropout 训练让模型不依赖单一模态；线上缺失用占位向量或按可用模态降级。

**Q5：怎么评估多模态召回有没有用？**
重点看**新品/长尾**的召回覆盖率和 CTR（整体指标会被头部稀释），以及冷启 item 的曝光起量速度。

---

## 七、选型一句话

- 新品多、长尾长、内容丰富(电商/短视频/图文) → 多模态召回价值大。
- 纯 id、内容贫乏的场景 → 收益有限。
- 落地优先级：先把内容向量喂双塔(性价比最高)，再考虑独立召回路。

---

## 八、监控指标

- 离线：Recall@K、新品/长尾子集 Recall、内容近邻人工抽检相关性。
- 在线：新品曝光起量速度、长尾占比、冷启 item CTR、整体 pvctr/uvctr。
