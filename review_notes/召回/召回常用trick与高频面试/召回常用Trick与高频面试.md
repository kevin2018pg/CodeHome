## 一、心法：召回和精排根本不是一回事

面试一上来最爱考的"认知题"，先把这层立住，后面所有 trick 都好解释。

- **目标不同**：精排要"排准"(pointwise/pairwise 打分排序)；召回要"不漏"(从全库捞回候选，高召回率 + 低时延)。
- **样本空间不同**：精排训练空间 ≈ 预测空间(都是曝光 item)；召回训练空间 ≠ 预测空间(线上面对**全库**)，所以**召回负样本主体必须是未曝光随机样本**，只用曝光未点击 = 大忌(SSB)。
- **结构约束不同**：召回要能 ANN 检索 → user/item 必须**解耦双塔**，不能像精排那样早期做特征交叉(否则 item 侧没法离线建索引)。

> 一句话：**召回是"在亿级里高效找回几百个不漏"，精排是"在几百个里排准"**。trick 基本都围绕"高召回 + 能索引 + 不被热门带偏 + 训得稳"。

## 二、样本类 Trick（最值钱，召回上限主要看这里）

| Trick | 干啥 | 备注 |
|---|---|---|
| **未曝光随机负为主体** | 治 SSB、学全库分布 | 召回负样本的地基 |
| **中段分数难负(~0.6 那批)** | 提精度、避 false negative | Facebook EBR：rank 100~500，最难的反而掉点 |
| **false negative 过滤** | 防教坏 | 剔历史正反馈 / 同 session 点过 / 高分 top |
| **曝光未点击当少量 hard 负** | 真实负反馈 | 要去噪：剔曝光<1s、广告位、历史转化过 |
| **curriculum 由易到难** | 训练更稳 | 初期多 easy，后期加 hard |
| **logQ 校正** | 防过度打压热门(纠偏) | in-batch 负偏热门，打分减 log Q(i) |
| **混合负采样 MNS** | 负样本分布贴近全库 | in-batch(热) + 全局均匀(长尾) |

> 完整体系(配比、甜区、Q 估计)见 [`负样本挖掘和召回效果评估`](../负样本挖掘和召回效果评估/负样本挖掘和召回效果评估.md)。

## 三、向量 / 结构 Trick

- **L2 normalize**：内积变余弦，度量统一、训练稳、压模长型热门偏置；配 temperature 用。**训练/serving 必须一致**(想保留模长热度先验或 MIND squash 时才不做)。
- **temperature τ**：InfoNCE 里 `cos/τ`，τ 小(0.05~0.1)拉大正负差、学得更"硬"，太小不稳。**τ 是召回最敏感超参之一**。
- **SENet / 特征加权**：双塔进塔前给特征自适应加权，缓解"塔太浅、交叉不够"。
- **多向量(MIND/ComiRec)**：一个用户出多个兴趣向量，治"单向量表达不了多兴趣"。
- **late interaction(Poly-encoder)**：保留少量交叉又能近似索引，介于双塔和单塔之间。
- **蒸馏**：用精排/单塔交叉模型当 teacher 蒸双塔，补双塔信息瓶颈。
> 双塔信息瓶颈与改进流派详见 [`U2I双塔召回`](../u2i双塔召回/U2I双塔召回.md)。

## 四、训练 Trick

- **sampled softmax / in-batch 负**：全库 softmax 算不起，用采样近似 + logQ 校正。
- **cross-batch / memory bank(MoCo 式)**：缓存历史 batch 向量扩大负样本规模，注意 stale，控队列长度 + 动量更新。
- **多目标/多行为**：点击+收藏+下单多行为联合，缓解单行为稀疏。
- **EMA / 双塔参数不对称更新**：item 塔更新慢一点更稳(看场景)。
- **位置/时间不泄漏**：序列召回严格 causal mask；时间特征不能用未来。

---

## 五、索引 / 工程 Trick

- **ANN 选型**：HNSW(召回率高、内存大) vs IVF-PQ(省内存、可量化、略掉点)；大规模用 IVF-PQ + 重排。
- **MIPS vs cosine**：norm 后用 cosine 索引；不 norm 走真 MIPS。两边度量必须和训练一致。
- **用户向量 & item embedding 版本对齐**：必须同模型版本同步更新，否则向量空间错位、掉点(高频工程坑)。
- **实时性**：用户向量小时级/实时更新抓即时意图，item 向量天级建索引；注意更新频率不一致带来的错位。
- **多路分数量纲对齐**：merge 前各路分数归一化(或用 RRF 按排名融合)。

---

## 六、热门 / 长尾 / 多样性 Trick

- **打压热门**(主动)：按热度负采样、打分减 `α·log(popularity)`、正样本 subsample、后处理热度配额/打散。
- **防过度打压**(纠偏)：logQ 校正还热门公道——先纠偏再按需打压，别混。
- **长尾扶持**：单独长尾/内容召回通路保供给；多样性配额。
> 两者区别与做法详见 [`负样本挖掘和召回效果评估`](../负样本挖掘和召回效果评估/负样本挖掘和召回效果评估.md) §七。

## 七、冷启动 Trick

- **新 item**：多模态/内容向量(图文视频 embedding)替代行为不足的 ID embedding，做语义泛化召回。
- **新用户**：默认向量/人群泛化向量 + 热门兜底；bandit(Thompson Sampling)探索。
- **ID/内容混合 embedding(EGES 思路)**：行为多用 ID，行为少用 side info，平滑过渡。
> 详见 [`多模态召回`](../多模态召回/多模态召回.md) 与 [`热门兜底与冷启动召回`](../热门兜底与冷启动召回/热门兜底与冷启动召回.md)。

---

## 八、多路融合 Trick

- **配额制**：每路给名额(MIND N1、双塔 N2、I2I N3…)，保多样性、防一路独大。
- **分数融合**：分数归一加权 或 **RRF(Reciprocal Rank Fusion，按排名而非分数)**，避免量纲不可比。
- **去重 & 打散**：merge 后 unique + 类目/作者打散，别让重排再把多样性去掉。

---

## 九、评估 Trick

- **离线**：Recall@K / HitRate@K / NDCG@K，进精排率。
- **判断一路有没有用**：**独占召回率/独占点击** + **减桶实验**(下掉它大盘掉多少)，别只看单路 Recall@K。
- **分桶占比**：按通路分桶看贡献 / 按流量分桶做 A/B。
> 详见 [`负样本挖掘和召回效果评估`](../负样本挖掘和召回效果评估/负样本挖掘和召回效果评估.md) §八。

## 十、高频面试题（带追问）

**Q1：召回和精排有什么本质区别？**
目标(不漏 vs 排准)、样本空间(全库 vs 曝光)、结构(必须解耦双塔可索引 vs 可任意交叉)。→ 引出"召回负样本为何不能只用曝光未点击"。
> 追问：召回为什么不能像精排那样做特征交叉？ → 交叉后 item 表示依赖 user，没法离线建 ANN 索引。

**Q2：召回负样本怎么选？为什么不能只用曝光未点击？**
主体未曝光随机(治 SSB) + 少量中段难负(提精度、避假负) + 可选去噪曝光未点击。只用曝光未点击 = 精排筛剩的好 item，训练空间≠预测空间。

**Q3：hard negative 为什么取中段(~0.6)而不是最难的？**
最难的(top)大概率是 false negative(用户其实喜欢只是没曝光)，会教坏模型；中段信息量大、假负可控。Facebook EBR 实证。

**Q4：in-batch 负采样有什么偏置？logQ 是干嘛的？**
热门 item 在 batch 出现多→被过度当负→被打压。logQ：打分减 `log Q(i)` 纠回无偏。
> 追问：Q(i) 怎么估？ → 流式频率估计(两次出现间隔)、count-min、全局计数。

**Q5：召回向量为什么要 L2 norm？什么时候不做？**
内积变余弦：度量统一、训练稳、压模长型热门偏置、配 τ。不做：真 MIPS 想保留模长热度先验、MIND squash 用模长表置信度。训练/serving 必须一致。

**Q6：temperature τ 起什么作用？太大太小会怎样？**
缩放 `cos/τ`。τ 小→正负拉得开、学得硬，太小→梯度不稳、过拟合难负；τ 大→区分不开。是召回最敏感超参之一。

**Q7：双塔的"信息瓶颈"是什么？怎么缓解？**
user/item 直到最后才内积，塔内无交叉，表达受限。缓解：SENet 特征加权、多向量、late interaction、用交叉模型蒸馏。

**Q8：召回偏热门怎么办？又要"防止过度打压热门"？矛盾吗？**
不矛盾：偏热门是结果偏置(主动压)，过度打压是采样副作用(logQ 纠偏)。先纠偏再按需打压。

**Q9：怎么评估一条召回通路有没有用？**
独占召回率/独占点击 + 减桶实验。只看单路 Recall@K 会高估(被别路覆盖也算命中)。

**Q10：线上召回指标涨了，下游 CTR 不涨，怎么排查？**
进精排覆盖率、merge 后重复率、分桶占比被稀释、重排是否把多样性去重掉、多路分数量纲是否可比。

**Q11：MIND 多兴趣 vs 序列召回 vs 双塔，怎么选？**
双塔(稳/快/基建)、序列 SASRec(即时意图/时效)、MIND(兴趣发散/多样性)。实践常并行互补。
> 多兴趣坍缩、动态路由等见 [`MIND多兴趣召回`](../mind多兴趣召回/MIND多兴趣召回.md)。

**Q12：I2I 的 ItemCF / Swing 区别？**
ItemCF 用共现，活跃用户/热门会虚高相似度；Swing 惩罚"共同点击两 item 的用户还点了一堆别的"(用户活跃度惩罚)，相似度更干净。

**Q13：冷启动怎么做？**
新 item 用多模态内容向量；新用户默认/人群向量 + 热门兜底 + bandit 探索；ID+内容混合 embedding 平滑过渡。

**Q14：多路召回怎么融合？**
配额 + 分数归一/RRF + 去重打散。注意各路分数量纲不可比，优先 RRF 按排名融合。

**Q15：ANN 索引 HNSW 和 IVF-PQ 怎么选？**
HNSW 召回率高但吃内存；IVF-PQ 省内存可量化但略掉点。超大规模常 IVF-PQ 粗筛 + 精排重算。

### 双塔专项六问（高频）

**Q16：了解混合负采样(MNS)吗？**
in-batch 负偏热门、长尾采不到(SSB)。MNS = **in-batch 负 + 全库均匀采样负**，均匀部分补长尾、拉近全库分布；两部分按各自采样概率分别 logQ 校正。

**Q17：in-batch negative 如何缓解对热门物品的过度打压？**
热门频繁进 batch 被当负 → 被打压。**logQ 校正** `score − logQ(i)`：热门 Q 大减得多，抵消惩罚，还原为对全库 softmax 的无偏估计。logQ 是纠偏(防过度打压)，不是主动压热门。

**Q18：双塔最后为什么加 L2 norm？**
内积=余弦(训练/检索度量统一)、压模长型热门偏置、数值稳定、多路分数可比；训练与 serving 必须都 norm。

**Q19：L2 norm + 温度为什么更好？**
只 norm 后内积挤在 [−1,1]、正负拉不开、梯度小；除以小 τ 放大差距、对比更锐、聚焦难样本、收敛快。norm 管稳定，τ 管区分锐度，配合用。τ 太小不稳、太大不分。

**Q20：双塔怎么缓解冷启动？**
新 item 用 side info + 内容/多模态 emb 替代没训好的 ID emb(+新品上采样/三塔隔离)；新 user 用画像泛化 + 实时 session + 人群兜底；配合 E&E 探索流量。

**Q21：SENet 和 LHUC 怎么加进双塔优化？**
两个**塔内增强**，都不破坏可索引性(塔内做、最后仍各出一个向量)。
- **SENet**(FiBiNET)：emb→MLP 间对**特征 field 自适应加权**——Squeeze(field pooling)→Excitation(两层 MLP+sigmoid 出权重，reduction ratio r)→Reweight。突出重要特征、压噪声，缓解信息瓶颈下"特征质量决定上限"。两塔均可加。
- **LHUC**(PPNet)：用**用户画像生成门控** `g`(`2*sigmoid`，范围 0~2 能放大能抑制)逐元素缩放隐层 `h⊙g`，做 per-user 个性化；gating 输入 **stop-gradient** 防污染主表示。主要加 user 塔。
- **区别**：SENet 管"哪些特征重要"(特征级)，LHUC 管"针对这个用户怎么调网络"(隐层级、个性化驱动)，可叠加。

## 十一、易错 / 陷阱题（面试官挖坑用）

- **"召回用 AUC 评估"** ❌ → 召回看 Recall@K/HitRate，AUC 是排序指标。
- **"负样本越难越好"** ❌ → 全 hard 会掉点(假负 + 不稳)，要 easy 为主 + 中段 hard。
- **"双塔可以做特征交叉提升效果"** ❌ → 早期交叉就没法建 item 索引；交叉只能放最后(内积)或用 late interaction 近似。
- **"L2 norm 训练做了，serving 图省事不做"** ❌ → 向量空间不一致直接掉点。
- **"曝光未点击是天然好负样本"** ❌ → 那是精排逻辑；召回会 SSB，且含位置偏差噪声。
- **"召回多样性做好就行，不用管重排"** ❌ → 重排去重可能把召回的多样性又抹掉，要全链路看。

---

## 十二、一句话总结

> 召回 trick 围绕四件事：**样本(未曝光为主+中段难负+logQ)**、**度量(L2 norm + temperature)**、**结构(解耦可索引 + 多向量/蒸馏补瓶颈)**、**链路(多路配额融合 + 独占贡献评估)**。面试主线永远是那句话——**召回是"在亿级里不漏地捞回几百个"，所以样本空间、负样本、可索引性、热门偏置，是它和精排最不一样、也最常被追问的地方。**
