# 召回负样本挖掘 和 召回效果评估 · 实战手记

> 视角：召回算法工程师。负样本是召回的命门——**模型上限很大程度由负样本质量决定，结构反而次要**。这套方法跨所有召回通路通用(双塔 / MIND / 序列 / 图)。
> 后半部分（§七、§八）讲**热门偏置的打压与纠偏**，以及**召回效果怎么评估（含分桶召回占比）**。

---

## 一、最重要的认知：召回 ≠ 精排的负样本逻辑

这是理解一切的前提，也是面试最爱考的点。

- **精排**：训练样本 = **曝光样本**(正=点击，负=曝光未点击)。精排线上只给召回送来的几百个 item 打分，**训练空间 ≈ 预测空间**，所以用"曝光未点击"当负样本是对的。
- **召回**：线上要从**全库(亿级)**里选。如果只用曝光未点击当负样本——那是精排筛剩下的"还不错的 item"，模型只学会"在好 item 里挑"，但线上要面对的是**全库里大量毫不相关的 item**。**训练空间 ≠ 预测空间**，这叫 **SSB(Sample Selection Bias，样本选择偏差)**。

> 结论：**召回负样本的主体必须是「未曝光的随机全库样本」**，让模型见过"全库长什么样"。只用曝光未点击是召回大忌。

---

## 二、负样本类型 & 各自"想干啥"

| 类型 | 怎么取 | 想干啥(目的) | 占比 |
|---|---|---|---|
| **随机/全局负**(easy) | 全库均匀或按频次采样(**未曝光**为主) | 学**全库分布**、建立基本区分力、**治 SSB** | 主体 |
| **in-batch 负**(easy) | batch 内别人的正样本互当负 | 工程高效的 easy 负；偏热门，要 logQ 校正 | 主体之一 |
| **hard 负**(难负) | 语义接近但用户没选(见 §三) | 磨**细粒度边界**、提精度 | 少量(10~30%) |
| **曝光未点击** | 曝光了没点 | 带**真实负反馈**的 hard 负，但有噪声 | 少量、要去噪 |

一句话：**easy 负管"全局能不能分开"(地基)，hard 负管"相近的能不能分细"(提精度)**，缺一不可，配比是关键。

---

## 三、Hard Negative 怎么挖（核心）

### 3.1 按召回排名取中段（最主流，"0.6 分数那批"）
用当前模型给 user 检索，**不取最顶部、不取尾部，取中段那批**当负样本(如 rank 100~500，或归一化分数 0.5~0.7)：

- **分数太高(top，0.9+)**：大概率是用户**真正相关**只是没曝光 → 当负样本就是 **false negative(假负)**，**会教坏模型**；
- **分数太低(0.1-)**：太 easy、没信息量；
- **中段(~0.6)**："看着像、但用户没选" → **信息量最大、假负风险可控**，这是难负的**甜区(semi-hard negatives)**。

> 经典依据：Facebook EBR 论文(Embedding-based Retrieval in Facebook Search)发现——**用"最难"的负样本反而掉点**，用排名中段(他们用 rank 101~500)效果最好。这就是"0.6 分数那批做负样本"的来历。

### 3.2 其它挖法
- **同类目/同属性未交互**：同类目但没点的，语义天然接近；
- **in-batch hardest**：batch 内和该 user 最相似的负样本挑出加权；
- **全局难负队列**：MoCo 式 memory bank 缓存历史 batch 向量，扩大难负规模(注意 stale，控队列长度)。

---

## 四、False Negative——挖难负最大的坑

难负挖得越狠，越容易把"用户其实喜欢、只是没曝光"的当成负样本。对策：
1. **取中段分数，避开 top**(见 §3.1)；
2. **过滤用户历史正反馈**(买过/收藏过/同 session 点过的剔除)；
3. **控制 hard 负比例**(easy 为主，hard 占 10~30%)；
4. **curriculum 由易到难**：训练初期多 easy、后期逐步加 hard，更稳。

> 一句话记住：**难负提精度 vs false negative 风险**，是负样本永恒的矛盾，甜区在中段分数。

---

## 五、曝光未点击：怎么用、怎么去噪

可以用，但**只作少量 hard 负补充**，且必须去噪(否则全是假负)：
- **历史转化/正反馈过的**剔除(曝光未点但其实喜欢)；
- **曝光时长 < 1 秒**剔除(根本没看到)；
- **广告位/特殊坑位**剔除(位置偏差导致的未点，不代表不喜欢)。

## 六、两个配套修正

- **logQ 校正(sampled softmax correction)**：in-batch 负偏热门(热门更常出现在 batch 被当负)，打分减 `log(采样概率 Q(i))` 抵消。Q(i) 用流式频率估计(全局计数 / 滑窗 / count-min)。
- **流行度 debias**：负采样按频次而非纯均匀，避免热门 item 主导，兼顾长尾覆盖。
- **混合负采样 MNS**：in-batch(偏热门) + 全局均匀采样(补长尾)，让负样本分布更接近全库。

---

## 七、热门偏置：打压热门 vs 防止过度打压（两件不同的事）

这俩名字看着矛盾，其实在**两个不同层面**，实际系统里同时做：

- **现象 A「召回偏热门 / 要打压热门」**：宏观**结果层面**——召回结果被头部热门 item 霸占，长尾出不来。根因是热门 item 正样本多、共现多、embedding 训练充分、内积天然大。这是**结果偏置**，需要**主动打压**。
- **现象 B「防止过度打压热门」**：是 in-batch / 按热度负采样这个**机制**带来的副作用——热门 item 频繁出现在 batch 里被当负样本，被**过度压制**，反而该出的热门出不来。这是**采样副作用**，需要**纠偏**还它公道。

> 不矛盾：一个是"结果太热 → 主动压"，一个是"采样机制误伤热门 → 纠回无偏"。正确顺序是**先用 logQ 纠偏到无偏，再按业务需要主动 debias 打压**。

### 7.1 打压热门怎么做（要抑制头部集中）

1. **负采样按热度加权**：负样本按 item 流行度采（popularity-sampled negatives），热门更常被当负样本，模型学会把"泛热门"往下压。
2. **打分减热度项**：score 减 `α·log(popularity)` 或加频率惩罚，直接削热门基础分（训练或 serving 都可加）。
3. **正样本对热门降采样**：word2vec 式 subsampling，热门 item 正样本按 `1 - sqrt(t/f)` 概率丢弃，减少热门在正样本里的主导。
4. **召回后处理/配额**：多路 merge 后做热度打散、提频控制、按热度分桶**限额**（头部桶设占比上限）。
5. **长尾/冷启专门通路**：单独通路保长尾覆盖，从供给侧补量（见 `热门兜底与冷启动召回`）。

### 7.2 防止过度打压热门怎么做（纠正采样误伤）

1. **logQ 校正**：`score_corrected = score - log Q(i)`，Q(i) 是 item i 被采为负的概率。热门 Q 大、减得多，正好抵消它被频繁当负的惩罚 → 还原成对全库 softmax 的无偏估计。
2. **准确估计 Q(i)**：流式频率估计（Google sampled softmax 用"两次出现间隔"在线估 Q）、count-min sketch、全局计数滑窗。**Q 估不准，纠偏就不准**。
3. **混合负采样 MNS**：in-batch（偏热门）+ 全局均匀，别让热门只靠 in-batch 被反复当负。
4. **正样本降采样别过狠**：热门正样本 subsample 过度本身也是一种过度打压；配比要盯"长尾覆盖率"和"热门召回率"两个指标一起调。

> 一句话：**logQ = "防过度打压"（纠正采样副作用、无偏化）；按热度负采样 / 减热度项 / 后处理配额 = "主动打压"（业务上要长尾）。先纠偏，再按需打压，别混为一谈。**

## 八、召回效果评估（离线指标 + 线上分桶占比）

### 8.1 离线指标

- **Recall@K / HitRate@K**：召回 TopK 里命中多少真实正样本，最核心。
- **NDCG@K / MRR**：考虑命中位置。
- **与精排的衔接**：召回 item 的进精排率、进精排后平均分，看召回质量。

### 8.2 线上"分桶召回占比"——有两种桶（面试易绕）

**(1) 按"召回通路"分桶（最常说的就是这个）**：每条通路（MIND / 双塔 / I2I / 热门 / …）算一个桶，看各桶贡献：

- **进精排占比**：每路召回 item 占进入精排集合的比例。
- **曝光/点击归因占比**：最终曝光、点击的 item 来自每条通路的比例（一个 item 常被多路召回，要先定归因口径：唯一归因 or 共享归因）。
- **独占召回率（unique contribution）**：某通路**独有、别路召不回**的 item 占比及其带来的独占点击——**这是判断一条通路有没有价值的关键**。
- 用途：判断这条通路是**真有用**，还是**被别的路覆盖/稀释了**。（MIND 坑5"分桶占比被稀释"指的就是这个。）

**(2) 按"流量/实验"分桶（A/B 桶）**：实验组 vs 对照组的流量切分，对比加/换某通路后大盘（CTR、时长、长尾覆盖）的变化。

### 8.3 怎么判断一条召回通路值不值得留

- **加桶/减桶实验**：上线/下线该通路，看大盘掉多少 = 它的真实边际贡献。
- **独占占比 + 独占点击**：占比高且有独占点击 = 有价值；占比高但点击全靠别路也能召回 = 冗余。
- 别只看单路离线 Recall@K——多路系统里"**边际贡献**"比"绝对召回率"更重要。

### 8.4 其它健康度指标

- **覆盖率**：召回覆盖的 item 库比例、长尾 item 占比。
- **多样性 / 热门集中度**：类目熵、基尼系数（看是否过度集中头部，呼应 §七打压热门）。
- **通路间重叠度**：两两 Jaccard，重叠太高 = 通路冗余。

## 九、常见问答（带追问）

**Q1：召回为什么不能只用曝光未点击当负样本？**
曝光未点击是精排筛过的"较好 item"，召回线上要面对全库，训练空间≠预测空间(SSB)。召回负样本主体必须是未曝光的随机全库样本。

**Q2：为什么 hard negative 取中段分数而不是最难的？**
最难的(top)大概率是 false negative(用户其实喜欢只是没曝光)，当负样本会教坏模型；最易的没信息量。中段(~0.6)信息量大且假负风险可控。Facebook EBR 实证 rank 中段最好。
> 追问·怎么界定"中段"？ → 经验区间(如 rank 100~500 / 分数 0.5~0.7)，需按业务调；配合过滤历史正反馈降假负。

**Q3：easy 负和 hard 负各起什么作用？怎么配比？**
easy(随机未曝光)建立全局区分、治 SSB，是地基、主体；hard(中段/同类目)磨细粒度边界、提精度，少量(10~30%)。全 hard 会掉点(假负+不稳)。

**Q4：in-batch 负采样有什么偏置，怎么修？**
热门 item 在 batch 出现多→被过度当负样本→被打压。用 logQ 校正(打分减 logQ)；并混合全局均匀采样补长尾(MNS)。

**Q5：曝光未点击能不能用？**
可作少量 hard 负，但要去噪：剔除历史转化过的、曝光<1s 的、广告位的。

**Q6：负样本太弱/太强会怎样？**
太弱(全 easy)→ 边界粗、向量易坍缩、召回虚高；太强(全 hard)→ 假负多、训练不稳、掉点。要 easy 为主 + 少量中段 hard + curriculum。

**Q7：又说"召回偏热门要打压"，又说"防止过度打压热门"，到底矛盾不矛盾？**
不矛盾，两个层面：偏热门是**结果偏置**(头部霸屏)要主动压(按热度负采样/减热度项/后处理配额)；过度打压是**采样副作用**(热门被频繁当 in-batch 负)要 logQ 纠偏。先 logQ 纠回无偏，再按业务主动打压。
> 追问·logQ 是打压还是不打压热门？ → logQ 是**纠偏**，把"被采样机制误伤的热门"还原成无偏；它不是主动打压，主动打压是另加的 debias。

**Q8：怎么判断一条召回通路有没有用？**
看**独占召回率/独占点击**(别路召不回的它能不能召回)，加做**减桶实验**(下掉它大盘掉多少)。只看单路 Recall@K 会高估——多路里要看边际贡献。

---

## 十、一句话总结

> 召回负样本 = **大量随机未曝光样本(治 SSB、学全库)** + **少量中段分数难负(提精度、避假负)** + **可选曝光未点击(真实负反馈、要去噪)**；核心矛盾是"难负提精度 vs false negative 风险"，甜区在**中段分数**；配合 logQ 校正与混合采样修偏置。
