# 二阶段兴趣点召回（兴趣点 → 内容 / 层次化多兴趣）· 实战手记

> 视角：召回算法工程师。这是和 MIND 并列的另一种**多兴趣范式**——不学隐式兴趣向量，而是显式"先选兴趣点(类目)、再选内容"。原型参考 QQ 短视频"多样性可控的二阶段召回通路"。
>
> 它和 MIND 是同一问题(多兴趣/多样性)的两种解法，建议和本目录的 MIND 笔记对照看。

---

## 项目介绍

> MIND 的多兴趣是软路由隐式学出来的向量，三个老问题一直没彻底解决：兴趣塌缩、不可解释、多样性不可控。我们做了一套显式的二阶段兴趣点召回通路来替换它。
>
> **核心思路**：把召回拆成两步——阶段一模型预测用户当前最感兴趣的 K 个兴趣点（类目），阶段二在每个兴趣点下做类目条件化的个性化 ANN 检索，最后按兴趣强度做等比 quota 分配合并候选集。兴趣点是显式的类目，天然不重叠，不存在塌缩问题，同时多样性变成可调的配额旋钮。
>
> **几个关键设计**：一是 quota 分配用等比缩减而不是等差或按概率直分，更贴合兴趣强度的指数衰减分布，防止单个兴趣吃满名额；二是头部兴趣信模型预估分，中长尾兴趣改用历史统计分，因为模型对长尾兴趣欠拟合，统计更可靠；三是在兴趣点层面引入负反馈（正向−负向预估重排），压制"一直推但从不点"的分发过头，粒度比内容级负样本更稳；四是额外建了一路兴趣点粒度的探索通路，先探方向再探内容，探索成本可控。
>
> 阶段一全量类目打分（类目空间小，几百~几千个，不需要 ANN），阶段二对海量内容用 ANN，两阶段天然分工。序列编码试过专门设计的 cate attention，实验发现没有打过标准 self-attention，最终用 self-attention。
>
> 上线后点击多样性（类目熵）提升明显，长尾内容曝光比例显著提升，相较于 MIND 可解释性强，运营可以直接干预兴趣点配额。

## 一、一句话讲清

把召回拆成两阶段(coarse-to-fine)：

1. **阶段一·选兴趣点**：模型预测用户的多个**兴趣点**(如内容类目)；
2. **阶段二·选内容**：在每个被选中的兴趣点下，再召回具体内容；
3. 线上一次性取多个兴趣点 + 各自召回，合并成候选集。

> 核心区别：MIND 的兴趣是**隐式学出来的 K 个向量**(会塌缩、不可解释)；这里的兴趣是**显式的兴趣点**(天然不重合、可解释、多样性可控)。

## 二、具体怎么做（端到端走一遍，重点看这节）

先把两个阶段各自的"输入→模型→输出"讲清，再讲怎么接起来，最后用一个短视频例子走一遍数。

### 2.1 阶段一：用户 → 兴趣点（这是一个"用户→类目"的预测模型）

- **要解决的问题**：给一个用户，预测他接下来会喜欢哪几个**兴趣点(类目)**。
- **输入**：用户画像(年龄/性别/地域…) + **行为序列**(最近看过的内容及其类目序列)。
- **模型**：行为序列过 self-attention(或 pooling)编码 → 得到用户表征 `u_base` → 接一个分类头。
- **输出**：对**全部兴趣点**(类目，通常几百~几千个，数量小)逐个打分 → 取 **Top-k 兴趣点 + 各自分数**。
- **关键点**：阶段一的"候选"是**兴趣点(类目)**，不是内容。类目空间小，所以**直接全量打分即可，不需要 ANN**（本质是一次矩阵乘法 `u_base @ category_emb.T`）。
- **训练**：label = 用户未来(下一刷/未来窗口)真实交互内容**所属的类目**。多分类用 sampled softmax，多标签用 BCE。
- （进阶）引入负反馈，线上按"正−负"差值给兴趣点重排(见 §4.3)。

**训练样本长什么样：**

每条样本 = 截止某时刻的历史序列 → 预测未来交互内容所属类目：

```
输入 X：用户画像(age=28, city=上海) + 行为序列[篮球A, 篮球B, 美食C, 游戏D]

# 方式一：sampled softmax（每条未来点击拆一条样本，label=类目ID）
样本1: X → y = 篮球(类目ID=5)
样本2: X → y = 游戏(类目ID=12)

# 方式二：BCE 多标签（未来窗口内所有点击类目聚合，label=0/1向量）
样本:  X → y = [0,...,1,...,1,...,0]  # 篮球=1, 游戏=1, 其余=0
```

和双塔/MIND 训练的核心区别：**label 是类目，不是具体内容 item**。

> 一句话：阶段一 = 把用户映射到"他要看哪几类"，输出是一串带分数的类目。

### 2.2 阶段二：兴趣点 → 内容（在选定类目下召回该用户爱看的内容）

要点：阶段二不是召回"该类目的泛热门"，而是召回"**该用户在这个兴趣点下**会喜欢的内容"。两种常见实现：

- **实现 A · 类目条件化向量检索(主力)**：把**目标兴趣点 embedding 拼进用户塔**，得到"该兴趣点下的用户向量" `u|c`，去**该类目的内容子库**(或全局库 + 类目过滤)做 ANN 取 TopN。本质是"带类目条件的双塔"。
  - `u|c` 怎么来：类目 embedding（和 item 侧共用同一套 embedding table，正常训练学出来）直接 lookup 取向量 `c`，与 `u_base` 拼接后过一层轻量 MLP：`u|c = MLP(concat(u_base, c))`。
  - 线上同一用户 K 个兴趣点各算一次 `u|c`，但 `u_base` 只算一次复用，K 次 MLP 计算量很小。
- **实现 B · 兴趣点下的个性化倒排/统计**：每个类目预先维护一个按(质量分 + 个性化分)排序的内容列表，直接取 TopN。**中长尾兴趣点常用这种**(配历史统计分，见 §4.2)。
- 阶段二也可以**直接复用已有的双塔 / I2I 通路**，只是加上"兴趣点(类目)约束"。

> 一句话：阶段二 = 在阶段一选出的每个类目里，做一次"类目内的个性化召回"。

### 2.3 两阶段怎么接起来（quota 分配是粘合剂）

1. 阶段一输出 k 个兴趣点 + 分数；
2. 按**等比缩减**(几何衰减)把总召回配额 N 分配给各兴趣点(头部多、长尾少，见 §4.1)；
3. 阶段二在每个兴趣点下，按分到的 quota 各召回若干内容；
4. 合并去重 → 候选集 → 进粗排/精排。

### 2.4 一个完整数值走查（短视频，秒懂）

> 设总召回配额 N=200，阶段一取 Top-5 兴趣点。

**阶段一**：输入用户最近 50 个观看视频序列 + 画像 → self-attention → 用户向量 → 对全部 500 个类目打分 → Top-5：

```
篮球 0.40 | 游戏 0.25 | 美食 0.15 | 数码 0.12 | 宠物 0.08
```

**quota 分配**(等比缩减，公比≈0.5 后归一到 200)：

```
篮球 ~100 | 游戏 ~50 | 美食 ~25 | 数码 ~13 | 宠物 ~6   （和≈200）
```

**阶段二**：对每个类目，用"用户向量 + 该类目条件"去对应内容库 ANN 取 quota 条：

```
篮球库取100 + 游戏库取50 + 美食库取25 + 数码库取13 + 宠物库取6
```

**合并去重** → 约 200 条候选，**多样性天然可控**(5 个类目都有，且按兴趣强度配比)。

### 2.5 训练与 serving 小结

- **训练**：阶段一(label=未来交互类目) 和 阶段二(label=未来交互的具体内容) 可分开训，也可共享序列编码底座(减少割裂，见 §6 坑2)。推荐共享底座：一个模型，阶段一接分类头、阶段二接检索头，联合训练或底座预训练后分头微调。
- **serving**：`u_base`（序列 attention）**只算一次** → 阶段一全量类目打分取 Top-k → 等比 quota → 阶段二 K 路 `u|c` 计算 + **ANN 并行检索**（K 路同时发出，耗时≈1路ANN）→ 合并。
- **索引**：阶段一**不用 ANN**(类目少，全量矩阵乘法)；阶段二**用 ANN**(内容海量)。这是两阶段在工程上的天然分工。
- **耗时参考**：用户塔推理 ~5-15ms，类目打分 <1ms，K路并行ANN ~5-20ms，整体 ~15-35ms，远低于召回层 SLA(50-100ms)。进一步优化：`u_base` 定期预算缓存到 Redis，线上直接读取跳过序列推理。

## 三、它为什么存在：精准打 MIND 的痛点

MIND 的多兴趣是软路由自动学的，带来三个老问题：**兴趣塌缩(向量重合)、不可解释、多样性不可控**。

二阶段范式直接**换赛道绕过去**：

- **不塌缩**：兴趣点是显式的(类目)，互不重合，不存在 MIND 的"K 个向量相似"问题；
- **可解释**："推这条内容，是因为模型预估你喜欢 X 兴趣点"；
- **多样性可控**：直接控制"选几个兴趣点、各分多少配额"，多样性变成一个可调旋钮。

> 谱系：MIND(隐式多兴趣) → 类目硬切(规则按类目拆) → **二阶段兴趣点召回(模型预测兴趣点 + 整套 quota/负反馈/探索)**。后者是这条线最成熟的形态——把"类目硬切"从规则升级成了可学习、成体系的范式。

## 四、核心设计（面试讲这些细节）

### 4.1 头部兴趣的 quota 分配

- 模型预估用户的 **k 个头部兴趣点**，按"越头部召回越多"分配名额。
- 分配策略试过多种：**borda 缩减、等差缩减、按预估概率值分配、等比缩减**——实践中**等比缩减(几何衰减)最好**。
- 直觉：头部兴趣强度衰减接近指数，等比配额比线性(等差)更贴合真实兴趣分布，又比"按概率直分"更稳(不会被个别高分兴趣吃掉所有名额)。

**等比缩减公式**（公比 r，K 个兴趣点，总配额 N）：

```
第 i 个兴趣点的权重：w_i = r^i   （i 从 0 开始，排名越高 i 越小）
quota_i = round(N × r^i / Σ r^j)  （j=0..K-1）

示例：N=200, K=5, r=0.5
权重：1, 0.5, 0.25, 0.125, 0.0625   sum=1.9375
配额：篮球~103 | 游戏~52 | 美食~26 | 数码~13 | 宠物~6
```

- r 取值范围通常 0.5~0.7：r 越小头部越集中，r 越大越均匀。
- 调参依据：在线监控类目熵（多样性指标），找到相关性和多样性的平衡点。

### 4.2 头部 vs 中长尾，用不同信号（重要洞察）

- **头部兴趣**：用**模型预估打分**选(模型对高频兴趣拟合好)。
- **中长尾兴趣**：试过"模型打分"和"用户历史流水统计分"，**统计分更好**——因为模型对长尾兴趣欠拟合，统计更可靠。中长尾兴趣点下用**均匀 quota**。
- 一句话：**头部信模型，长尾信统计**。这是个很真实、很能体现经验的判断。

### 4.2.1 兴趣选择的另一条路：纯统计（LongRetriever 实践）

二阶段选兴趣点有两条路：**模型预测打分**（我们的方案）和**纯统计**（LongRetriever，字节电商）。两者各有适用场景：

**纯统计方案——"Random in Top"（LongRetriever）**：

```
Score(c_i) = Σ_{行为 b ∈ 用户全身历史} sign(b 属于类别 c_i) / 距今天数
```

对每个类目用时间加权的历史行为频次打分（越近的行为权重越高），然后：
1. 取 **Top-M 类目**（M=20）；
2. 从 Top-M 里**随机抽 N 个**（N=5）用于本次请求。

**为什么随机抽而不直接用 Top-5**：字节实验数据对比——

| 策略 | 重叠率（AER，越低越好） | 独占贡献（UER，越高越好） | CTR×CVR |
|---|---|---|---|
| Random 5 in Top 20 | **18.7%** | **9.3%** | 7.9% |
| 直接 Top 5 | 30.4% | 7.2% | **8.3%** |

直接 Top-5 转化效率略高，但和已有召回路高度重叠，独占贡献低，加了和没加差不多。Random in Top 引入探索，独占贡献高，系统层面收益更大。**这和我们方案里单独建探索通路的思路一致，只是实现更简单。**

**两种方案对比**：

| 维度 | 模型预估（我们方案）| 纯统计（LongRetriever）|
|---|---|---|
| 头部兴趣准确性 | 高（模型拟合好）| 高（统计也可靠）|
| 长尾兴趣准确性 | 低（模型欠拟合）| **高（统计更可靠）**|
| 工程复杂度 | 高（需要训练分类头）| **低（纯计数）**|
| 可解释性 | 中 | **高（直接看频次）**|
| 探索能力 | 需单独探索通路 | 随机 Top-M 内置探索 |

结论：**长尾兴趣信统计这个判断在这里再次得到验证**。业务早期/工程资源紧张时，纯统计 + Random in Top 是更务实的起点。

### 4.3 兴趣点级别引入负反馈

负向 label 定义：用户**曝光未点击**内容所属的类目（可进一步按行为强度加权：快速 skip > 普通未点 > 停留未点）。

**两种实现方式：**

**方式一：一个头，正负 label 混合训练（简单，常用）**

```python
# 同一个分类头，label 里正负样本都有
# 点击类目 label=1，曝光未点类目 label=0，未曝光类目 mask 掉
loss = BCE(logits, labels, mask=exposed_mask)
```
负反馈通过负样本直接融进训练，模型自己学会"曝光多但不点的类目 → 打低分"。线上直接用输出分排序即可。

**方式二：两个头，正向/负向分开预估（可控性强）**

```python
p_pos = sigmoid(u_base @ W_pos)   # 点击类目 label 训练
p_neg = sigmoid(u_base @ W_neg)   # 曝光未点类目 label 训练
loss = BCE(p_pos, click_label) + α * BCE(p_neg, skip_label)

# 线上：差值重排
score = p_pos - β * p_neg
```

优势是正负解耦，β 可独立调节惩罚力度（运营可实时干预）；代价是多一套参数、训练稍复杂。

**效果示例：**
```
          p_pos   p_neg   差值(β=1)
篮球       0.40    0.05    0.35   ← 真喜欢，排第一
游戏       0.35    0.30    0.05   ← 一直推但不点，被压到最后
美食       0.15    0.02    0.13
```

- 作用：压制"系统分发过头"——你一直被推某类但从不点，负反馈把它压下去。
- 巧妙处：负反馈用在**兴趣点排序层**，而不是内容负样本，粒度更高、聚合信号更稳。

### 4.4 序列模块（含一个诚实的负结果）

- 试过传统 **self-attention** 和为二阶段定制的 **cate attention**(类目感知注意力)；
- 结果 **self-attention 反而更好**。

**cate attention 的思路**：既然阶段二要在特定类目下检索，attention 计算时让同类目 item 之间权重更高。具体做法有三种：加类目偏置 `score(i,j) += λ·I(cate_i==cate_j)`、mask 掉跨类目的 attention、或把类目 embedding 拼进 Q/K。

**为什么没打过 self-attention：**
1. **跨类目信息被切断**：用户从篮球跳到游戏这个"兴趣跳转"本身有信号，cate attention 把跨类目 attention 压低，丢失了这部分；
2. **self-attention 已能自学类目聚合**：同类目 item embedding 本就相近，普通 attention 天然会给同类目分配更高权重，无需手动约束；
3. **强归纳偏置反而限制了模型**：用户跨类目兴趣（如摄影+旅游）被类目 mask 切断；
4. **类目体系本身有噪声**：类目粒度粗细不一，cate attention 对这些噪声更敏感。

- 面试价值：敢讲"我设计的定制结构没打过 baseline"这种负结果，反而显得有判断力、不吹。

### 4.5 兴趣点探索（E&E）

- 用兴趣点打分的**泛化性**找"最该被探索的兴趣点"，单独建一路**多样性探索通路**；
- 对应通用 E&E，但做在**兴趣点粒度**而非内容粒度——探索成本更可控(先探兴趣方向，再探内容)。

## 五、和 MIND / 类目硬切的对比


| 维度    | MIND(隐式多兴趣)  | 类目硬切(规则) | 二阶段兴趣点召回                |
| ----- | ------------ | -------- | ----------------------- |
| 兴趣怎么来 | 软路由自动学 K 个向量 | 规则按类目切   | **模型预测兴趣点**             |

| 可解释   | 弱            | 强        | 强                       |

| 跨类目兴趣 | 保留           | 丢失       | 取决于兴趣点定义                |
| 复杂度   | 中            | 低        | **高(两阶段+quota+负反馈+探索)** |
| 长尾处理  | 一并学(欠拟合)     | 看桶       | **头部信模型/长尾信统计**         |

## 六、常见坑

**坑 1 · 两阶段误差累积**
现象：阶段一兴趣点选错，阶段二再准也白搭。
解：阶段一适当放宽(多召几个兴趣点) + 阶段二兜底；监控阶段一兴趣点准确率单独评估。

**坑 2 · 阶段一/阶段二割裂(分开训有 gap)**
原因：两阶段独立训练，目标不完全一致。
解：联合训练 / 阶段二的反馈回传阶段一 / 共享序列底座 / 用统一 label 对齐。

**坑 3 · 兴趣点体系强依赖**
和类目硬切同病：兴趣点粒度(类目体系)定义不好，整条链路受限；跨类目/细粒度兴趣表达受限。
解：兴趣点体系要专门设计、可迭代；必要时多粒度兴趣点。

**坑 4 · quota 分配不当**
全给头部→多样性差；过度均匀→相关性差。
解：等比缩减折中，并按场景调衰减系数；多样性指标在线监控。

**坑 5 · 长尾兴趣点噪声**
模型对长尾欠拟合，硬用模型分会乱。
解：长尾改用历史统计分 + 均匀 quota(见 §4.2)。

## 七、常见问答（带追问）

**Q1：二阶段兴趣点召回和 MIND 比，好在哪？**
MIND 兴趣是隐式向量，会塌缩、不可解释、多样性难控；二阶段用显式兴趣点(类目)，不塌缩、可解释、多样性可控(quota 旋钮)。本质是"换范式"绕过 MIND 塌缩。

> 追问·那它有什么代价？ → 强依赖兴趣点体系、两阶段误差累积、复杂度高(quota/负反馈/探索一整套)、阶段间可能割裂。

**Q2：阶段一到底预测什么、要不要 ANN？**
预测用户对**全部类目**的偏好分，取 Top-k 兴趣点。类目只有几百~几千个，**全量打分即可，不用 ANN**。ANN 只在阶段二(海量内容)用。

**Q3：阶段二怎么保证召回的是"该用户在该类目下爱看的"，而不是类目泛热门？**
把**目标类目作为条件**拼进用户塔，得到"类目条件化的用户向量"再去检索(实现 A)；长尾类目则用个性化倒排/统计分(实现 B)。

**Q4：头部和长尾兴趣为什么用不同信号？**
头部兴趣模型拟合好→用模型预估分；长尾兴趣模型欠拟合→用历史流水统计分更可靠，并配均匀 quota。"头部信模型、长尾信统计"。

**Q5：quota 怎么分？为什么等比最好？**
按"越头部召回越多"分配，试过 borda/等差/按概率/等比，**等比(几何衰减)最好**——更贴合兴趣强度的指数衰减，又比按概率直分更稳(防单兴趣吃满名额)。

**Q6：负反馈怎么用进来的？**
阶段一对每个兴趣点同时预估正向/负向倾向，线上按 `正−负` 差值重排兴趣点，压制"一直推你却不点"的分发过头。用在兴趣点粒度而非内容负样本。

**Q7：cate attention 没打过 self-attention 说明什么？**
说明为二阶段硬造的定制结构不一定有效，简单 self-attention 已能很好编码序列。能讲这种负结果体现实验素养(敢承认没用的尝试)。

**Q8：兴趣点探索为什么放在兴趣点粒度？**
先探"该不该给你某个兴趣方向"，再在该方向探内容——探索成本可控、不容易因探索伤体验，比直接在海量内容上探索更稳。

## 八、选型一句话

- 要**多样性可控 + 可解释 + 不怕塌缩** → 二阶段兴趣点召回(尤其内容/短视频，类目即兴趣点)。
- 要轻量、隐式自动发现兴趣 → MIND/ComiRec。
- 业务先验极强、要最简实现 → 类目硬切。
- 三者其实是同一问题的"从隐式到显式、从简单到体系化"的谱系，按团队成本和场景选。
