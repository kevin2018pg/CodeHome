# MIND 多兴趣召回 · 实战手记

> 视角：召回算法工程师。酒店/旅行推荐场景，经历了 **DSSM 双塔 → MIND v1 → MIND v2（Hard Negative + 正交正则 + K 自适应）→ 360 工业化改进** 的完整迭代。
> 参考：[360多兴趣召回 MIND 实战经验分享](https://zhuanlan.zhihu.com/p/678879375)

---

## 项目介绍

> 原来用双塔召回，用户兴趣多样但只有一个向量表示，召回结果"三不像"，多样性差。引入 MIND 做多兴趣召回，主要做了三块优化：
>
> **模型结构上**：把 softmax 软路由改成 argmax 稀疏硬分配，每个行为只贡献一个胶囊，从机制上防止兴趣塌缩；胶囊初始化用 kmeans++ 策略，基于预训练 item embedding 选 K 个分散的起点，收敛更快；删掉冗余变换矩阵 S，squash 换 L2-norm，和整个向量体系对齐。
>
> **Embedding 设计上**：行为序列用预训练 embedding 并 fix（stop gradient），让路由聚的是稳定语义兴趣，不被 loss 短路；item 塔相同初始化但可学习，两套解耦。
>
> **负采样上**：幂律分布采样控制热门打压力度（α 可调），冷门 item 设基础先验频率防翘尾；三层难负体系 + logQ 校正，难负在 MIND 里还兼治兴趣塌缩。
>
> 最终 Recall@50 提升 8%，线上 GMV +1.2%。

## 一、一句话讲清解决什么

双塔用**一个向量**表示用户，把所有历史行为压成一个兴趣点。酒店场景：一个用户可能同时有**周末亲子短途、长途商务、境外度假**三种兴趣，单向量是它们的"平均"，ANN 检索时离每个簇中心都偏，召回结果"三不像"，多样性差、长尾覆盖不到。

**MIND 的做法**：用户侧从 1 个向量升级成 **K 个兴趣向量**，每个向量代表一类兴趣，线上 K 路并发 ANN 检索 TopN，merge 去重。本质是「多塔用户侧」，和双塔不冲突，可以并存兜底。

---

## 二、模型结构（能口述就行）

1. **Embedding 层**：item_id / 类目 / 品牌等稀疏特征 → embedding。
2. **Multi-Interest Extractor**：Dynamic Routing 把行为序列自动聚成 K 个兴趣胶囊（soft k-means）。
3. **Label-aware Attention（仅训练）**：用 target item 把 K 个兴趣软选成 1 个向量，让梯度精准流向最相关兴趣。
4. **Sampled Softmax / InfoNCE Loss**。
5. **线上 Serving**：跳过 Label-aware，K 个兴趣向量各自 ANN 检索 → merge → 候选集。

> 训练/推理不对称：训练有 target 所以能"选兴趣"；推理没有 target（target 就是要找的），直接 K 路覆盖。

---

## 三、核心模块

### 3.1 Dynamic Routing

本质是**可微的 soft k-means**，把行为序列自动聚成 K 堆。迭代固定 3 次：

1. 对每个行为的 K 个路由分数做 softmax，得到软分配权重 `w[N,K]`；
2. 加权聚合得候选兴趣向量 `z_k = Σ_i w[i,k] · e_i`；
3. squash / L2-norm 得兴趣向量 `v_k`；
4. 更新路由 logit `b[i,k] += e_i · v_k`——item 和哪个胶囊越像，下轮路由越多 → 正反馈自组织聚类。

**为什么 3 次够**：第 1 次均匀、第 2 次初步分化、第 3 次基本收敛，再多收益极小还不稳。

**MIND 相对原始 CapsNet 的三处关键改造**：

1. **共享变换矩阵 S**（原版每对 (行为,兴趣) 独立矩阵，亿级存不下；共享后兴趣和 item 落同一空间，才能 ANN）；
2. **routing logit 随机初始化**（全零则 K 个胶囊更新对称 → 退化成单兴趣，随机初始化打破对称）；
3. **动态 K**（行为越多 K 越大，新用户 K=1~2，避免给低活用户无中生有）。

### 3.2 Squash 与 L2-norm

```
squash(z) = (‖z‖²/(1+‖z‖²)) · (z/‖z‖)   # 保方向、压模长到(0,1)
L2-norm(z) = z / ‖z‖                       # 保方向、模长=1
```

胶囊原版用 squash，模长表"兴趣置信度"、方向表"兴趣语义"，所以不能用逐元素的 ReLU/tanh（会破坏方向）。工业改进用 L2-norm：全模型向量都归一化，squash 的"模长=置信度"语义失效，统一换 L2-norm 更简洁，还消灭了 `z/‖z‖` 趋零时的数值不稳定（上线必须加 eps）。

### 3.3 Label-aware Attention

```
scores = V_u · e_t           # [K]，每个兴趣和 target 内积
a = softmax(scores ^ p)       # p 控注意力软硬
v_u = a · V_u                # [d]，K 个兴趣压成 1 个用户向量用于算 loss
```

**p 的作用**：p=0 退化成平均（多兴趣失效）；p→∞ 退化成 hard selection（只有最相关兴趣拿到梯度，其他欠训练）。论文 p=40，工程从 p=20 起调，盯各兴趣激活分布是否均匀。

### 阶段 0：双塔基线（DSSM）

用户侧行为序列 mean pooling → 1 个向量。问题明确：单兴趣，召回多样性差，长尾酒店覆盖率低，多种偏好用户体验差。

---

### 阶段 1：MIND v1 上线验证

**做了什么**：用户侧换 Dynamic Routing，K=3；Redis 用户向量 `[1,D]` → `[3,D]`，小时级近线更新；3 路并发 ANN，merge 取 max score。

**发现的问题**：

- **兴趣塌缩**：3 个兴趣向量两两余弦相似度 > 0.9，K 路召回大量重叠，等于白做多路；
- **短序列用户变差**：行为 < 5 的用户每个胶囊只有 1~2 条行为，全是噪声，召回比双塔还差。

**根因**：soft k-means 没有任何约束逼 K 个胶囊分开，负样本太简单时模型不分化兴趣也能降 loss，塌缩是结构性问题，不是调参能解决的。

---

### 阶段 2：v2 系统优化（主要功劳）

**针对兴趣塌缩**：

① **Hard Negative 体系重建**——MIND 里难负有双重价值：提精度 + 治塌缩（负样本太简单时，不分化兴趣也能降 loss）。三层体系：


| 层级         | 来源                   | 目的                |
| ---------- | -------------------- | ----------------- |
| L0 随机负（主体） | 全库未曝光采样 + in-batch 负 | 治 SSB，让模型见全库分布    |
| L1 中段难负    | 同城市曝光未点击（正:负≈1:1）    | "看着像但没选"，信息量大     |
| L2 最难（少量）  | 同城市+同价位+同评分段         | 精细区分，但不能全用（易引入假负） |


配比：easy 为主 + 难负占 10~30%；curriculum 由易到难，更稳、塌缩更少。
过滤假负：历史购买过的 / 曝光 < 1 秒的 / 广告位曝光的。

② **正交正则**——直接惩罚 K 个兴趣向量两两余弦相似度：

```
L_orth = (1/(K*(K-1))) * Σ_{i≠j} G[i][j]²    G = V_norm · V_normᵀ
总损失 L = L_rec + λ * L_orth     λ 从 0.01 起调
```

等价于逼 Gram 矩阵趋近单位阵。调参信号：λ 太小还塌缩，太大兴趣被强行推开 Recall 掉，盯两两余弦均值（目标 0.3~0.6）。

**针对短序列**：

③ **K 自适应**：行为 <5 → K=1 走双塔兜底；5~~15 → K=2；15~~30 → K=3；>30 → K=4。
④ **损失换 InfoNCE**（τ=0.07），加流行度修正，防热门城市酒店被系统性压低。

**v2 结果**：兴趣向量余弦相似度 0.92 → 0.61，多样性继续涨，短序列指标恢复正常。

### 阶段 3：工业化打磨（360 实战改进）

参考 [360 MIND 实战分享](https://zhuanlan.zhihu.com/p/678879375)，在 v2 基础上做了六处工程化简化与优化，进一步提升稳定性和效果：

**① 删矩阵 S**

原版 embedding+S 两个可学习参数串联是冗余的，等价于让 embedding 直接对齐目标维度 d。推荐场景行为位置不重要（和 NLP 不同），不需要 per-pair 不同变换。直接删掉 S，简化参数，效果等价。

**② squash → L2-norm**

全模型都用 L2-norm 归一化（embedding 出来就归一化，item 向量也归一化），squash 的"模长=置信度"语义用不到（L2-norm 后模长全为 1）。统一换 L2-norm：更简洁、消灭数值不稳定、训练与 ANN 度量一致（内积=余弦）。

**③ 稀疏路由 + 每轮覆盖（最关键，治塌缩最釜底抽薪）**

原版 softmax 软分配，每个行为同时贡献所有胶囊，兴趣差异弱。改法：

- **w 稀疏化**：argmax 硬分配，每个行为只保留最大兴趣权重=1，其余置 0，每行为只贡献一个胶囊，兴趣间信息流完全切断；
- **logit 每轮覆盖**（不累加 `b += e·v`，改为直接用 `e·v` 覆盖），防路由惯性，每轮完全基于当前兴趣向量重新判断。

和正交正则的区别：正交正则是**事后惩罚**（塌缩了再推开）；稀疏路由是**从机制切断**（根本不让信息流到无关兴趣），更根本。

**④ 两套 Embedding + Stop Gradient**

```
Input Embedding Layer (fix)         ← 行为序列用这个，预训练初始化后冻结
         ↕ Same Init
Output Embedding Layer (learnable)   ← label-aware + loss 用这个，正常更新
```

fix 的核心逻辑：路由靠行为 embedding 的相似度来做分配，如果 embedding 随 loss 更新，模型会走捷径——挪动 embedding 靠近 target 而不是真正分化兴趣。fix 强迫模型只能靠真正分化胶囊来降 loss，路由语义更稳定、可解释。预训练来源：item2vec 或已有双塔 item 塔结果。

**⑤ 幂律负采样 + 冷门先验频率 L**

```
P_sample(i) = max(freq(i), L)^α / Σ_j max(freq(j), L)^α
```

α 控打压力度（越大对热门打压越重，从 α=0.75 起调）；L 给冷门 item 保底频率，防止超冷门 item 极少被当负 → 推理时胶囊把它排很靠前（"翘尾"现象）。L 设为全库频率均值的 1% 左右。

**⑥ kmeans++ 胶囊初始化**

随机初始化胶囊可能聚在一起，路由要花很多轮才分开。改用 kmeans++ 策略选 K 个分散的起点：

1. 随机选一个 item embedding 作为第 1 个胶囊；
2. 对每个剩余 item，算它与已选胶囊集合的最短余弦距离 d(i)；
3. 按 d(i)² 概率采样下一个（距离越远被选概率越大）；
4. 重复到 K 个。

maxmin 变体更简单：每次直接选距离最大的（确定性，更极端，可能踩噪声点）。两者都保证初始胶囊语义尽量分散，配合 fix 的预训练 embedding 效果更稳。

### 改进路线总览


| 阶段     | 核心问题                | 解法                            | 效果信号            |
| ------ | ------------------- | ----------------------------- | --------------- |
| v0→v1  | 多样性差                | 引入 MIND，K=3 多路 ANN            | 多样性涨，但塌缩暴露      |
| v1→v2  | 兴趣塌缩 + 短序列差         | Hard Negative + 正交正则 + K 自适应  | 余弦相似度 0.92→0.61 |
| v2→工业化 | 路由不稳 + 冷热 item 采样失衡 | 稀疏路由、两套 emb、幂律采样、kmeans++ 初始化 | 收敛加速，推理分布更均衡    |

## 五、训练与落地要点

**样本格式**（Spark 离线构造，写 parquet）：

```json
{"user_id":"u_123","profile":{...},"hist":[{"item_id":1001,"cat":"亲子酒店","action":"order","ts":1717200000},...],"target_item_id":2001,"label":1}
{"user_id":"u_123","profile":{...},"hist":[...],"target_item_id":5099,"label":0}
{"user_id":"u_123","profile":{...},"hist":[...],"target_item_id":3012,"label":0,"hard_neg":true}
```

关键：user 侧冗余存（不 join，直接读）；target 只存 id、特征训练时查表保证最新；hist 严格时序防穿越；in-batch 负训练框架动态生成，不预存。

**全局负采样**：维护最近 7 天曝光 item 采样池（50~100 万），幂律采样；全局负里 30% 补低频长尾防热门偏；in-batch 负打底配 logQ 校正（Q(i) 用滑窗计数或 Google 间隔估计法，无需精确归一化）。

**用户向量推理**：


| 场景             | 策略                                                |
| -------------- | ------------------------------------------------- |
| 行为实时性要求高       | 线上实时 forward（新行为立即生效）                             |
| 延迟预算紧/GPU 资源有限 | 离线预计算 + Redis 缓存（T+1 或小时级）                        |
| 两者兼顾           | **热用户实时 forward、冷用户走缓存**，Redis miss 时 fallback 实时 |


**塌缩监控**：每日抽样 10~100 万活跃用户批量 forward，算两两余弦均值，p90 > 0.95 触发告警。监控 job 和线上必须用同一模型版本（版本错位会假告警）。

## 六、高频面试问题（滴水不露版）

### 【主题一】MIND 原理与设计

**Q：Dynamic Routing 和 k-means 有什么关系和区别？**

本质上 Dynamic Routing 就是 **soft k-means 的可微版本**，但有三处关键区别：

1. **可微、端到端**：k-means 是离线聚类，无法反向传播；Dynamic Routing 嵌入网络，兴趣向量和 embedding 一起梯度更新；
2. **软分配**：k-means 每个点硬分到一个簇；Dynamic Routing 用 softmax 软分配，每个行为按比例分给所有兴趣（工业改进改成 argmax 硬分配）；
3. **度量是内积而不是欧氏距离**：`b[i,k] += e_i · v_k`，用向量内积衡量亲和度，和 ANN 检索用的度量一致。

**Q：为什么 routing logit 不能全零初始化？**

全零时，K 个胶囊的 softmax 权重完全相同 → 聚合出来的 K 个候选兴趣向量完全相同 → 更新 `b += e·v` 对所有兴趣加的量也完全相同 → K 个胶囊始终对称，永远不会分化。本质和神经网络权重不能全零初始化一样——对称性破坏不了，梯度更新方向完全一致，多个"神经元"退化成一个。随机初始化打破对称，每个胶囊从略微不同的起点出发，自然分化。

**Q：squash 换 L2-norm 两者数学一样吗？为什么可以换？**

**不一样**：squash 把模长压到 (0,1)、模长有"兴趣置信度"的语义；L2-norm 直接把模长变为 1，丢弃了模长信息。能换的前提是：整个模型全用 L2-norm 归一化体系（embedding 出来就归一化，item 向量也归一化），此时 squash 的"模长=置信度"语义根本用不到——模长永远是 1，没有区分度。在这个前提下，squash 是多余的，换 L2-norm 更简洁，还消灭了 `z/‖z‖` 当 z 趋零时的数值不稳定问题。

**Q：Label-aware Attention 训练时为什么要聚成 1 个向量？线上为什么不用？**

训练：loss 是"用户表示 vs target item"的内积对比，必须是 1 个用户向量。如果 K 个兴趣平均再算 loss，梯度会散给所有兴趣，无论哪个兴趣负责都拿到一样的梯度信号，互相干扰、兴趣不收敛。Label-aware 用 target 的相关性选出最应负责的兴趣，梯度精准流向它，其他兴趣该学什么继续学什么。

线上：没有 target（target 就是我们要找的东西），无法做 label-aware 选择。直接保留 K 个兴趣向量各自独立检索，正是多兴趣的核心价值——K 路覆盖不同兴趣簇，多样性比单向量好。类比 Dropout——训练时的辅助信号，推理时去掉。

**Q：为什么要 fix input embedding / 两套 embedding 的设计？**

动态路由靠行为 embedding 之间的相似度来做分配（`b += e_i · v_k`）。如果行为 embedding 随 loss 梯度更新，模型会找捷径：不用真正学会分化兴趣，只需把不同用户的行为 embedding 靠近各自的 target，路由就能"假装"分化。fix 切断这条捷径，强迫模型只能靠真正改变路由逻辑来降 loss，兴趣分化才是真实的。

item 塔 Output Embedding 没有路由依赖，正常更新才能学到召回任务的精细匹配。两者相同初始化保证训练初期的向量空间对齐，label-aware 的内积有意义。

### 【主题二】兴趣塌缩

**Q：兴趣塌缩的根因是什么？如何系统性解决？**

**根因**：Dynamic Routing 本身没有任何"让 K 个胶囊分开"的约束。当负样本太简单时，模型把 K 个兴趣胶囊都映射到同一个"大众偏好方向"也能有效区分正负样本、降低 loss。所以塌缩是结构性问题，不是参数没调好。

四种解法（从软到硬）：


| 方案               | 原理                   | 优势           | 局限             |
| ---------------- | -------------------- | ------------ | -------------- |
| Hard Negative    | 难负逼模型必须分化兴趣才能区分正负    | 兼治精度和塌缩，一举两得 | 构建成本高，假负需过滤    |
| 正交正则             | 直接惩罚两两余弦相似度          | 算力几乎不增，效果可控  | 事后惩罚，治标不治本     |
| 稀疏路由（argmax 硬分配） | 每行为只贡献一个胶囊，从机制切断信息流  | 釜底抽薪，比正则更根本  | 梯度离散，训练略不稳定    |
| 类目硬切             | 按类目拆行为子序列，每路只学一个类目兴趣 | 基本消灭塌缩，可解释强  | 丢失跨类目兴趣，依赖类目粒度 |


我们用的是 Hard Negative + 正交正则 + 稀疏路由三层叠加，余弦相似度从 0.92 降到 0.61。

**Q：K 路全被一个兴趣占满怎么处理？**

先区分是**塌缩**（K 个向量高度相似）还是**配额问题**（向量不同但 ANN 一路覆盖太多）。前者用上面四种方案治；后者是工程问题——merge 时对每路设 TopN 配额，防止一个兴趣占满最终候选集，同时 pow_p 调小让梯度更均匀分给所有兴趣。

### 【主题三】负采样

**Q：负样本怎么构建？曝光未点击能直接用吗？**

曝光未点击**不能作主体**。它是精排筛剩的"较好 item"，只用它会引入 SSB（Sample Selection Bias）——训练分布偏向精排 top 区间，和线上全库分布差异极大，模型在真实全库上泛化性差。主体必须是**未曝光随机负**，让模型见全库分布；曝光未点击只作中段难负补充，提供"看着像但没选"的精细区分信号。

**Q：in-batch 负的热门偏置问题怎么解决？**

in-batch 负是 batch 内其他正样本的 item。正样本采样本身有热门偏置（热门 item 被正样本采到的概率高），所以 in-batch 负里热门 item 出现更频繁 → 模型把热门 item 当负样本打压更重 → 推理时热门 item 相关性被系统性低估。

解法：**logQ 校正**，修正训练打分：`score'(u,i) = u·v_i - log Q(i)`，Q(i) 是 item i 的采样概率。热门 item Q 大，减去 log Q 后被打压的幅度减小，还原真实相关性。Q(i) 工程实现：滑窗计数（维护最近 100 万样本的 item 频次，用 count-min sketch 省内存）或 Google 间隔法（记录两次出现的时间间隔，Q≈1/平均间隔，只存上次出现时间，内存极省且感知分布漂移）。Q 无需精确归一化，相对频率即可。

**Q：难负为什么取召回排名中段而不是最难的 top？**

取 top 最难的负样本很可能是**假负样本**（用户实际感兴趣但没曝光到的 item）。Facebook EBR 论文实验表明：取召回排名中段（如 rank 101~500）而非最靠前的效果最好——够难（模型能从中学到有效区分），但不至于太难引入大量假负，打乱正确的排序关系。

### 【主题四】K 的选择与短序列

**Q：K 怎么选？踩过什么坑？**

K 没有普适最优值，和场景行为量强相关。我们的策略是 K 自适应：

- 行为 <5 条 → K=1，走双塔兜底（行为太少撑不起多兴趣，强行 K=3 每个胶囊只有 1~2 条行为=噪声，反而变差）；
- 5~~15 条 → K=2；15~~30 条 → K=3；>30 条 → K=4。

K 不能无限大：K 大兴趣冗余（相邻的向量越来越难分化）、训练不稳定、推理延迟线性增加。同时要监控各兴趣的激活分布——如果某个兴趣几乎没有行为路由进去（激活度极低），说明 K 过大，有"空胶囊"问题。

**Q：短序列用户 MIND 效果差怎么处理？**

三板斧：① K 自适应，短序列直接走双塔兜底；② 多行为类型合并（点击+收藏+下单+搜索）扩充有效序列长度；③ side info 辅助（用户画像里的历史兴趣标签作为软先验引导路由）。短序列的本质问题是每个胶囊的行为支撑不足，任何单纯调路由参数的方案都治标不治本，根本解是"让行为够用"或"降低对行为数量的要求"。

### 【主题五】工程问题与线上排查

**Q：离线 Recall@K 涨但线上 CTR 不涨，怎么排查？**

按链路逐层拆：

1. **进精排覆盖率**：MIND 召回的 item 有多少进了精排？占比太低 = 召回质量有问题（相关性不够），精排也没处发力；
2. **K 路 merge 后 unique 增幅**：< 10% 说明塌缩严重，K 路覆盖高度重叠，多路没发挥作用；
3. **分路独占贡献**：MIND 在全部召回路里的独占贡献（只有 MIND 召回到、其他路没召回到的 item 占比），独占太低说明被其他路稀释，加了等于没加；
4. **多路 merge 量纲**：MIND 的 ANN 距离分数和其他召回路的分数是否可比？量纲不对会导致 MIND 的 item 在 merge 排序中系统性靠后；
5. **重排去重（最容易忽视）**：召回阶段做了多样性，但重排/精排有"相似去重"或"同类目限制"逻辑，把 MIND 召回的多样 item 又过滤掉了，召回的努力在重排被白白抹掉。

**Q：用户向量和 item embedding 版本错位怎么处理？**

现象：模型更新后线上指标掉点，但离线验证正常。根因：用户向量和 item embedding 用了不同版本的模型生成，两个向量空间不对齐，ANN 检索的距离失去意义。

解法：模型更新时必须原子性地同步刷新用户向量和 item embedding，用同一版本的模型前向推理生成，不能一个先一个后。工程上可以用版本号标记，线上检索时校验两个向量的版本号是否一致。

**Q：MIND 的 item 塔能复用已有双塔的 item index 吗？**

**不能**。MIND 有自己的 item embedding（Output Embedding Layer，可学习），和标准双塔的 item 塔是两个完全独立的模型训练出来的，向量空间不对齐。MIND 必须用自己的 item 塔离线批量刷全库 item embedding，建独立的 ANN 索引。两个模型可以共享 item 特征工程（item 特征表），但向量必须各自刷。

## 七、监控指标

- **离线**：loss 波动（±3σ 阻断）、Recall@K、各兴趣向量两两余弦均值（塌缩告警阈值 > 0.95）；
- **在线**：进精排覆盖率、K 路 merge unique 增幅、各兴趣路独占贡献、pvctr/uvctr 同环比、兴趣塌缩监控（日抽样批量 forward）。
