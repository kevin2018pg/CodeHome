# MIND 多兴趣召回 · 实战手记

> 视角：召回算法工程师。酒店/旅行推荐场景，经历了 **DSSM 双塔 → MIND v1 → MIND v2（Hard Negative + 正交正则 + K 自适应）** 的完整迭代。
> 本笔记按「面试怎么讲」组织，不做科普。

---

## 一、一句话讲清它解决什么

双塔用**一个向量**表示用户，把所有历史行为压成一个兴趣点。

我们的场景是酒店：一个用户可能同时有**周末亲子短途、长途商务出行、境外度假**三种兴趣。单向量是这几个兴趣的"平均"，ANN 检索时离每个簇中心都偏，召回结果"三不像"，多样性差、长尾酒店覆盖不到。

**MIND 的做法**：用户侧从 1 个向量升级成 **K 个兴趣向量**，每个向量代表一类兴趣，线上每个向量独立 ANN 检索 TopN，merge 去重。本质是「**多塔用户侧**」，跟双塔不冲突，可以并存兜底。

> 一句话总结（面试结尾用）：**多兴趣是结构（K 个向量），Label-aware 是训练选路（梯度精准流向对的兴趣），ANN 是线上提速（K 路并发检索）。**

---

## 二、结构从下到上（能口述就行）

1. **Embedding 层**：item_id / 类目 / 品牌 / 用户画像等稀疏特征 → embedding。
2. **Pooling 层**：每个行为 item 的多字段 embedding 拼接/池化，得到单个行为表示。
3. **Multi-Interest Extractor（核心）**：Dynamic Routing 把行为序列**自动分成 K 堆**，输出 K 个兴趣胶囊向量。本质是 soft k-means。
4. **Label-aware Attention（只训练用）**：用 target item 选出最相关的那个兴趣，聚合成 1 个向量算 loss。
5. **Sampled Softmax / InfoNCE Loss**。
6. **线上 Serving**：跳过 Label-aware，直接拿 K 个向量做 K 路 ANN → merge → 候选集。

> **训练/推理不对称**是高频追问点：训练有 target 所以能"选兴趣"聚成 1 个向量；推理没有 target（target 就是要找的东西），所以直接用 K 个向量各自检索。类比 Dropout——训练时的辅助机制，推理时不要。

---

## 三、核心模块（讲原理够用就行，别背公式）

### 3.1 Dynamic Routing（胶囊路由）

本质：**soft k-means**，把行为序列自动聚成 K 堆。迭代固定 3 次：

- 算软分配权重 `w = softmax(b)`（每个行为分配到各兴趣的权重）；
- 加权聚合得到候选兴趣向量 `z = Σ w·e`；
- squash 压一下 `v = squash(z)`；
- 更新路由 logit `b += e·v`——**item 和哪个胶囊越像，下轮就越多路由过去**，正反馈形成自组织聚类。

**为什么 3 次够**：第 1 次均匀分配、第 2 次初步分化、第 3 次基本收敛，再多收益极小、计算翻倍还不稳。

**相对原始 CapsNet 的 3 处改造（这是阿里 MIND 的精髓，面试爱深挖）**：

1. **共享变换矩阵 S**：原版每对 (行为, 兴趣) 一个矩阵，item 量级根本存不下；共享后兴趣和 item 落同一空间，才能直接点积/ANN。
2. **routing logit 随机初始化**（不是 0）：初始化为 0，K 个胶囊会更新完全对称 → 收敛成同一个向量 → 多兴趣退化成单兴趣。随机初始化打破对称（和"神经网络权重不能全 0"同理）。
3. **动态兴趣数 K**：行为越多 K 越大，新用户只给 1~2 个，避免给低活用户"无中生有"造空兴趣。

### 3.1.1 路由细节拆解（新手必懂）

**① b 是什么 / 怎么把行为分到各兴趣**
b 是"路由打分矩阵"(routing logit)，形状 `[N, K]`：N=行为数，K=兴趣数。`b[i][j]` = 第 i 个行为属于第 j 个兴趣的**原始亲和度打分**。对每个行为在它的 K 个兴趣分上做 softmax，就得到权重 `w[i][:]`（和为 1 的分布）：

```
        兴趣1  兴趣2                  兴趣1  兴趣2
行为1 [ 2.0,  0.1 ] --softmax--> [ 0.87, 0.13 ]  → 主要属于兴趣1
行为2 [ 0.2,  1.8 ] --softmax--> [ 0.17, 0.83 ]  → 主要属于兴趣2
行为3 [ 1.5,  1.4 ] --softmax--> [ 0.52, 0.48 ]  → 两边都沾
```

所以是**软分配**(每个行为按比例分给所有兴趣)，不是硬分到某一类。

**② 行为序列 / 权重数量**

- 每个行为 = 用户一次交互的 item，embedding 成向量 `e_i`；
- 序列工程上 padding 成**定长**(取最近 N 个，不足补齐+mask，超过截断)，算法本身支持变长；
- 长度 20、K 个兴趣 → b、w 都是 `[20, K]` 矩阵 → 共 **20×K 个权重**(每个行为对 K 个兴趣各一个)，不是 20 个。

**③ 加权聚合**
兴趣 j 的向量 = 所有行为向量(经共享矩阵 S 变换)按"分给兴趣 j 的权重"加权求和：

```
z_兴趣1 = 0.87·e₁ + 0.17·e₂ + 0.52·e₃
z_兴趣2 = 0.13·e₁ + 0.83·e₂ + 0.48·e₃
```

行为越属于某兴趣(w 越大)，对该兴趣向量贡献越大 → 把行为"聚"成不同兴趣中心。

**④ logit 是什么**
logit = 没过 softmax/sigmoid 之前的**原始打分**(来自统计的 log-odds)。CTR 里 sigmoid 前的值、多分类 softmax 前的那排分、路由里的 b，都是 logit。过 softmax/sigmoid 后才变成 0~1 的概率/权重。

**⑤ 原始 CapsNet 是什么**
Hinton 团队 2017《Dynamic Routing Between Capsules》提出的胶囊网络，本是做**图像识别**的(神经元输出从标量变向量、动态路由替代池化)。MIND 把它的动态路由借来做多兴趣聚合。其中"共享变换矩阵 S"是 MIND 相对原版的关键改造：原版每对(行为,兴趣)一个独立矩阵，item 亿级存不下；共享 S 后兴趣和 item 落同一空间，才能内积/ANN 召回。

### 3.2 Squash 为什么不用 ReLU/tanh

核心区别是**作用范围**：ReLU/tanh 是**逐元素**(每维单独变换，会改方向)；squash 是**整个向量级**操作——保方向、只压模长：

```
squash(z) = (‖z‖² / (1 + ‖z‖²)) · (z / ‖z‖)
             └──── 压缩模长到(0,1) ───┘  └ 单位方向(不变) ┘
```

胶囊用**模长表"兴趣置信度"、方向表"兴趣语义"**。ReLU/tanh 逐维操作会把方向和模长语义都搞乱；squash 只压模长不动方向，正好契合胶囊设计。

> 坑：`z/‖z‖` 分母有 `‖z‖`，z 趋零时数值不稳定，**必须加 eps**，否则 loss NaN。

### 3.3 Label-aware Attention

训练时有 target item，K 个兴趣里只有最相关那个该为它负责。如果 K 个向量平均了再算 loss，梯度会散给无关兴趣、互相干扰。

**操作三步（设 embedding 维 d、兴趣数 K；兴趣向量 V_u 形状 [K,d]，target 向量 e_t 形状 [d]）**：

```
Step1 相关性:  scores = V_u · e_t        # [K,d]·[d] → [K]，每个兴趣和target的内积
Step2 pow+softmax:  a = softmax(scores ^ p)  # [K] → [K]，注意力权重（p 在这里起作用）
Step3 加权聚合:  v_u = a · V_u            # [K]·[K,d] → [d]，K个兴趣压成 1 个用户向量
```

维度流：

```
行为序列 [N,d] --动态路由--> K个兴趣 [K,d]
target item                      e_t [d]
  V_u·e_t → scores [K] → pow(p)+softmax → a [K] → a·V_u → v_u [d]
  训练: v_u[d] 与 正/负样本[d] 算内积 → sampled softmax loss
```

**为什么聚合成 1 个、而不是保留多个**：这一步**只在训练发生**。训练有 target，loss 是"1 个用户表示 vs 这个 target"，所以要先从 K 个兴趣里软选出和 target 最相关的那个、聚成 1 个向量再算 loss，让梯度精准流向该负责的兴趣（平均则梯度被无关兴趣稀释）。**线上没有 target，不聚合——直接保留 K 个兴趣向量各自 ANN 检索**。这就是 MIND 训练/推理不对称的核心。

**p 在哪起作用**：Step2 里给相关性打分加的指数 `softmax(scores^p)`，控制注意力软硬：

- **p→0**：`scores^0=1`，权重相等 → 退化成平均，多兴趣失效；
- **p→∞**：最大分主导 → hard selection，只有最相关兴趣拿到权重≈1，其它≈0（其它兴趣几乎拿不到梯度，欠训练）；
- 论文用 **p=40** 偏 hard，工程上我从 20 开始调，看各兴趣激活分布是否均匀。

## 四、版本迭代（面试主线，按这个讲故事）

### v0 · DSSM 双塔基线

- 用户侧：行为序列 mean pooling → 1 个向量。
- 问题：单兴趣，召回多样性差，长尾酒店覆盖率低。

### v1 · MIND 上线验证（K=3）

- 用户侧换成 Dynamic Routing，K=3；Redis 用户向量从 `[1,D]` 扩到 `[3,D]`，小时级近线更新；线上 3 路并发 ANN，merge 取 **max score**。
- **结果**：召回多样性涨，但 CTR 提升有限。定位到两个问题：
  - **兴趣塌缩**：3 个兴趣向量余弦相似度 > 0.9，等于白做多路；
  - **短序列用户反而变差**：行为 < 5 的用户，每个胶囊只有 1~2 条行为，全是噪声。

### v2 · Hard Negative + 正交正则 + K 自适应（主要功劳在这版）

**① Hard Negative 分级（最值钱的一块）** — 完整体系见 `[负样本挖掘和召回效果评估.md](../负样本挖掘和召回效果评估/负样本挖掘和召回效果评估.md)`

> MIND 里难负有个**特殊作用**：不只是提精度，更是**治兴趣塌缩**——负样本太简单时，模型不分化兴趣也能降 loss，K 个向量就塌成一团；难负逼模型必须把兴趣分开才能区分。所以 MIND 对负样本质量比双塔更敏感。

**负样本分层（easy 打底 + 中段难负提质，不是越难越好）**：

- **L0 easy 负（主体）**：从**全库随机/未曝光**采样(治 SSB，让模型见全库分布)；工程上用 in-batch 负打底(1:1000+，自动)，配 logQ 校正抵消热门偏置。
- **L1 中段难负（甜区）**：同城市曝光未点击(正:负≈1:1)——"看着像但没选"，信息量大。
- **L2 最难（少量）**：同城市+同价位+同评分段(1:1)。**注意别全用最难的**：太难易引入 false negative、反而掉点(Facebook EBR 结论：取召回排名**中段**而非 top 最好)。

**为什么不只用曝光未点击**：曝光未点击是精排筛剩的"较好酒店"，只用它会 SSB(训练空间≠线上全库)。所以**主体必须是未曝光随机负**，曝光未点击只作中段难负补充。

**关键是过滤假负样本**（追问必问）：

- 用户历史**购买过**的酒店（曝光未点击但其实喜欢）；
- 曝光时长 **< 1 秒**（根本没看到）；
- **广告位**曝光（位置偏差导致的未点击，不代表不感兴趣）。

**配比与课程**：easy 为主 + 难负占 10~30%；curriculum 由易到难，训练更稳、塌缩更少。

**② 兴趣正交正则**：惩罚 K 个兴趣向量两两余弦相似度，强制它们分化。

**③ K 自适应**：行为 <5 → K=1（直接走双塔兜底）；5~~15 → K=2；15~~30 → K=3；>30 → K=4。

**④ 损失从 sampled softmax 换 InfoNCE**（τ=0.07），加流行度修正，防止热门城市酒店被系统性压低。

- **v2 结果**：兴趣向量平均余弦相似度 **0.92 → 0.61**，多样性继续涨，短序列用户召回指标恢复正常。

## 五、训练与落地要点（速记）

- **K 怎么选**：行为越长 K 越大，但太大冗余/不稳。我们 3 起步 + 自适应。大库也有人 4/8/16 起步。
- **负采样是命门**：召回效果一大半由负样本质量决定。**未曝光随机负打底(治 SSB) + 中段难负提质(治塌缩、避假负) + logQ 校正**，缺一不可。MIND 比双塔更吃负样本质量(难负还兼治塌缩)。完整体系见 `[负样本挖掘和召回效果评估.md](../负样本挖掘和召回效果评估/负样本挖掘和召回效果评估.md)`。
- **序列长度**：长序列截断/分桶（防不同兴趣搅混）；短序列调小 K 或直接兜底。
- **线上合并**：K 路结果去重 + 业务规则过滤 + **每路配额**（防一个兴趣占满）。merge 取 **max** 不取 sum（避免热门 item 被多兴趣命中虚高）。
- **监控**：Recall@N、覆盖率（unique items）、兴趣多样性、兴趣向量两两相似度（塌缩预警）、线上 CTR/GMV 链路指标。

## 六、面试官最爱问（优缺点）

**优点**

- 多兴趣表示，解决一人多兴趣的表征冲突；
- K 路检索覆盖广、多样性好、长尾捞得回来；
- 工程友好，天然适配向量索引 ANN。

**缺点（要主动说，显得你踩过坑）**

- 对训练/采样更敏感，负样本和分布很影响结果；
- K 选择敏感：太小不够分，太大冗余/不稳；
- 短序列用户收益小，兴趣拆分反而像噪声。

## 七、常见坑位与排查清单（实战核心）

**坑 1 · K 路全被一个兴趣占满**
现象：K=3 但三个向量几乎一样，召回高度重叠。
解：每路配额强制多样性；监控向量相似度 >0.95 告警；pow_p 调小让梯度更均匀流到所有兴趣。

**坑 2 · 兴趣向量全局塌缩（重合度高）**
现象：所有用户的兴趣向量在空间里扎堆、两两余弦相似度高（训练层面的全局问题），K 路召回大量重叠。
原因：软路由没有任何约束逼 K 个胶囊分开，模型不分化兴趣也能降 loss。
四种解法（从软到硬，详见 §九 进阶对比）：

1. **Hard Negative**：负样本变难，逼模型必须分化兴趣才能降 loss；
2. **正交正则**：直接惩罚 K 个兴趣向量两两余弦相似度（推荐首选，便宜有效）；
3. **换 ComiRec-SA**：多头注意力 + 可学习 query，天然比胶囊路由更易分化；
4. **类目硬切（Hard-Search）**：按业务类目把行为序列拆成多个不相交子序列，每路只学一个类目的兴趣——从源头消灭塌缩，纯净度/独立性最强，代价是丢了自动发现兴趣和跨类目兴趣的能力。

**正交正则具体怎么做（可直接讲的实现步骤）**

1. 拿到用户的 K 个兴趣向量 `V = [v1, …, vK]`，每个维度 d。
2. **先 L2 归一化**每个 `vk`（这样内积=余弦，不被模长干扰，很关键）。
3. 算两两相似度矩阵 `G = V_norm · V_normᵀ`，形状 `K×K`，`G[i][j]` 就是兴趣 i、j 的余弦。
4. 对角线是自己和自己（=1），不罚；只罚**非对角元素**。损失：

```
L_orth = (1 / (K*(K-1))) * Σ_{i≠j} G[i][j]^2
```

   也可写成 `||G - I||_F^2`（G 减单位阵的 Frobenius 范数平方），等价含义：逼 G 趋近单位阵 = 兴趣两两正交。
5. 总损失 `L = L_rec + λ * L_orth`，`λ` 一般 **0.01~0.1**，从小往大调。
6. 调参信号：λ 太小→还塌缩；太大→兴趣被强行推开、单兴趣表达变差、Recall 掉。盯**两两余弦均值**（目标降到 0.3~0.6）和 Recall@K 一起看。

> 一句话原理：动态路由本身不含"让兴趣分开"的约束，模型不分化也能降主损失；正交正则给它加了一个"分开才不被罚"的外力。比 Hard Negative 更直接、算力几乎不增。

**坑 3 · 负采样太弱，召回虚高**
现象：离线指标好，线上 item 质量差(且 K 个兴趣易塌缩)。
解：未曝光随机负打底 + 中段难负(避开 top 防假负)；过滤 in-batch 假负样本(同 batch 内有真实交互的不能当负)；logQ/流行度修正。详见 `[负样本挖掘和召回效果评估.md](../负样本挖掘和召回效果评估/负样本挖掘和召回效果评估.md)`。

**坑 4 · 短序列用户效果差**
解：K 自适应；行为不足直接走双塔兜底保下限；多行为类型（点击+收藏+下单）合并增加有效序列长度。

**坑 5 · 线上指标涨、下游 CTR 不涨（最容易忽视）**
排查清单：

1. **覆盖率**：MIND 召回的 item 有多少进了精排？占比太低=召回质量有问题。
2. **重复率**：K 路 merge 后 unique item 增幅，<10% 说明塌缩严重。
3. **分桶占比**：MIND 在所有召回路里的占比、独占贡献，是不是被其他路稀释了（分桶占比/独占召回率的完整口径见 `[负样本挖掘和召回效果评估.md](../负样本挖掘/负样本挖掘和召回效果评估.md)` §八）。
4. **重排检查**：多样性 item 是不是在重排被相似去重逻辑又过滤掉了（召回做了多样性、重排一去重就白费）。
5. **分数量纲**：多路 merge 时 MIND 的分数和其他路是否可比。

**坑 6 · 用户向量和 item embedding 版本错位（工程坑）**
现象：模型更新后掉点。
原因：用户向量小时级更新、item embedding 天级更新，两个向量空间对不上。
解：模型更新时用户向量和 item embedding 必须用**同一个模型版本**同步更新。

## 八、常见问答（背熟，带追问）

**Q1：MIND 和双塔什么区别？**
双塔 1 个用户向量，MIND K 个。核心改动在用户侧塔——Dynamic Routing 把行为聚成 K 个兴趣胶囊，线上 K 路 ANN merge 去重。本质是「多塔用户侧」，可和双塔并存（双塔兜底、MIND 提多样性）。

> 追问·K 怎么生成的？ → 胶囊网络动态路由，soft k-means，迭代 3 次软分配 + 加权聚合，相似的行为越来越聚到同一胶囊。

**Q2：训练为什么要 Label-aware Attention，线上为什么不用？**
训练有 target，用它把梯度精准导向最相关的兴趣，避免梯度散给无关兴趣互相糊。线上没有 target，直接 K 路各自检索覆盖多兴趣。训练/推理不对称，类似 Dropout。

> 追问·pow_p 怎么选？ → 论文 40 偏 hard，工程从 20 调；太小退化成平均、多兴趣失效，太大只训一个兴趣、其他欠训练。看各兴趣激活分布是否均匀。

**Q3：squash 为什么不用 ReLU/tanh？**
胶囊模长有语义（兴趣置信度），ReLU/tanh 按维度独立操作会破坏它。squash 对整个向量操作，方向不变只压模长到 (0,1)。

**Q4：负采样怎么做？曝光未点击一定是负样本吗？**
分层：**未曝光随机负打底**(治 SSB) + in-batch 负(1:1000+，配 logQ 校正抵消热门) + **中段难负提质**(同城市/同价位曝光未点击，1:1，取召回排名中段而非最难)。MIND 里难负还**兼治兴趣塌缩**。

> 不一定是真负：要过滤①历史购买过的②曝光<1秒的③广告位曝光的（位置偏差）。别只用曝光未点击(那是精排筛剩的，召回用会 SSB)。完整体系见 `[负样本挖掘和召回效果评估.md](../负样本挖掘和召回效果评估/负样本挖掘和召回效果评估.md)`。

**Q4.1：为什么 MIND 比双塔更吃负样本质量？**
负样本太简单时，模型不分化兴趣也能降 loss → K 个兴趣向量塌成一团。中段难负逼模型把兴趣分开才能区分，所以难负在 MIND 里**既提精度又治塌缩**。

**Q5：K 怎么选？踩过什么坑？**
酒店行为 10~~50 条，K=3 起步。短序列(<5) K=3 反而变差（每胶囊 1~~2 条行为=噪声），改 K 自适应：<5 走 K=1+双塔兜底，5~~15 K=2，15~~30 K=3，>30 K=4。

**Q6：兴趣向量高度相似怎么排查？**
先看两两余弦相似度 >0.95=塌缩。方向：①负样本太简单→加 Hard Negative；②加正交正则；③pow_p 调大；④动态路由换 ComiRec-SA（多头注意力，可学习 query 分化更好）；⑤业务先验强的话直接类目硬切（见 §九）。

> 追问·如果直接按类目硬切，那还算 MIND 吗？ → 严格说是用业务先验替代了软路由，更像"按类目拆多路召回"。在类目=强意图信号的场景（鞋服电商）很划算、可解释、能涨点；但丢了自动发现兴趣和跨类目兴趣的能力，属于"简化式优化"。更稳的折中是把类目当路由的辅助监督/偏置（类目引导路由），既压塌缩又保留学习能力。

**Q7：线上指标涨但 CTR 不涨怎么排查？**
按链路拆：覆盖率 → merge 后 unique 增幅 → 多路 merge 量纲是否可比 → 重排是否把多样性 item 又去重掉了。最后一个最容易忽视。

**Q8：MIND 和 ComiRec 区别？**
兴趣提取方式不同：MIND 用胶囊动态路由；ComiRec 有两个变体——ComiRec-DR（类似 MIND）和 **ComiRec-SA**（多头自注意力，K 个可学习 query 对行为序列做 attention）。工业上 SA 更常用：训练稳、梯度顺、没有 squash 数值问题。

## 九、进阶：软路由 vs 硬切 vs 类目引导路由（治塌缩的三条路线）

> 这是面试的高阶亮点。当被问到"兴趣塌缩/重合度高你还有什么更狠的招"时，能讲出这三条路线的取舍，比只会说"加正交正则"高一个段位。

**问题根因**：原版 MIND 的兴趣是软路由（soft k-means）**自动学**的，没有任何约束逼 K 个胶囊分开 → 容易塌成一团、每个兴趣里还混多类目（重合度高、类目分布不集中）。

**三条路线对比**


| 路线                  | 做法                                      | 塌缩   | 跨类目兴趣  | 可解释 | 灵活性 | 适用              |
| ------------------- | --------------------------------------- | ---- | ------ | --- | --- | --------------- |
| ① 软路由 + 软约束         | 动态路由不变，加 Hard Negative / 正交正则 / 调 pow_p | 缓解   | 保留     | 弱   | 高   | 通用，先验弱的场景       |
| ② 类目硬切（Hard-Search） | 按类目把行为拆成不相交子序列，每路只学一个类目兴趣，K=类目数         | 基本消灭 | **丢失** | 强   | 低   | 类目=强意图的场景（鞋服电商） |
| ③ 类目引导路由（折中）        | 仍用软路由，但把类目当辅助监督/先验偏置去引导                 | 显著缓解 | 保留     | 中   | 中   | 想兼顾稳定与灵活        |


**② 类目硬切（业界真实案例）**

- 思路：把行为序列按类目（如 运动鞋/休闲鞋/服装/包配/其他）解耦成多个子序列，强制模型在各自类目空间内学兴趣向量，保证纯净度与独立性。K 固定 = 类目数。
- 效果（某鞋服电商）：DPV +1.14%、PVCTR +0.85%。
- **本质**：用业务先验替代软路由——从"让模型自动发现兴趣"改成"人为规定兴趣=类目，每类目单独一路召回"。
- **代价**：丢自动发现兴趣 + 跨类目兴趣（如"马拉松备战"=鞋+服+补给被拆碎）；强依赖类目粒度；"其他"桶不纯净；空桶/稀疏桶要兜底。
- **被怼预案**：被问"那还算 MIND 吗 / 为啥不直接每类目一个双塔"——答：它确实更接近"按类目拆多路召回"，但在类目=强意图信号的场景里，简单、可控、可解释、能涨点，是典型"能上线 > 优雅"的工程选择；而且它其实暴露了原版路由在该场景没学出有意义兴趣的事实，所以干脆用先验。

**③ 类目引导路由（我更推荐的折中）**

- 不硬切，而是把类目作为**路由的辅助监督 / 先验偏置**：让动态路由在类目引导下学。
- 好处：既压住塌缩、保住纯净度，又**保留跨类目兴趣和自动发现能力**。
- 代价：更复杂、更难调。工程上如果"糙快猛"就够用，②反而更划算。

**一句话结论**：

> 选哪条取决于**业务先验有多强**。先验强（类目即意图）→ 硬切最划算；先验弱 / 要跨类目 → 软约束或类目引导路由。硬切的"好"是工程意义上的好（稳定+可解释+能上线），不是模型能力意义上的好。

## 十、和竞品怎么选（一句话决策）

- 召回要多样性 + 用户兴趣发散 → **MIND**（一人多向量，覆盖广）。
- 精排/粗排要细粒度用户-物品交互 → **DIN/DIEN/DCN**（单次打分强，但要 target，召回用不了）。
- 召回只求快 + 简单可控 → **双塔 DSSM**（最工业化、最好部署）。
- 用户兴趣不发散（垂类单一）→ 双塔/序列模型够用，MIND 收益有限（K 个向量分化不出来）。

## 十一、监控指标体系（加分项）

**离线（卡上线）**

- loss 低于过去 30 天 90 分位 → 可能过拟合/兴趣不发散，阻断；
- loss 高于过去 30 天最大值 ×1.1 → 没训好、召回太散，阻断；
- loss 不在 (均值 ±3σ) → 阻断。

**在线**

- 召回覆盖率（进精排占比）、K 路 merge unique 数、各兴趣向量两两相似度（塌缩预警）、pvctr/uvctr 同环比告警。
