# I2I 召回（Item-to-Item）· 实战手记

> 视角：召回算法工程师。I2I 是工业界**最古老也最稳**的通路，"看了又看/买了又买"的底座。覆盖协同类(ItemCF/Swing)和向量类(i2i embedding)两大流派。

---

## 一、一句话讲清

给定用户近期交互过的 item（trigger），找出和它**最相似/最相关**的一批 item 召回。相似度可以来自**共现统计**（ItemCF/Swing）或**向量近邻**（item embedding ANN）。

> 和 U2I 的区别：U2I 用"用户向量"检索，I2I 用"物品"当触发点。I2I 解释性强（"因为你看了 A 所以推 B"）、对新用户友好（只要有 1 个行为就能触发）。

---

## 二、两大流派

### 2.1 协同过滤类（基于共现统计）

**ItemCF**：两个 item 被同一批用户交互得越多越相似。
- 相似度 `sim(i,j) = 共现(i,j) / sqrt(|U_i|·|U_j|)`（余弦形式）。
- 问题：**热门 item 和谁都相似**（活跃用户什么都点）。

**Swing（淘宝经典，重点）**：用"用户对"来度量相似，惩罚活跃用户。
- 直觉：如果用户 u、v **都**交互了 i 和 j，且 u、v 共同交互的 item 越少（越"专一"），这对 (i,j) 的相似信号越可信。
- 公式核心：`sim(i,j) = Σ_{u,v∈共同交互i,j} 1/(α + |I_u ∩ I_v|)`。
- 优点：有效压制热门、抗噪，比 ItemCF 质量高很多，是工业 I2I 的主力。

**用户活跃度惩罚**：活跃用户贡献的共现权重降权（`1/log(1+|I_u|)`），防"什么都点的人"污染相似度。

### 2.2 向量类（i2i embedding）
- item embedding（来自双塔 item 塔 / 序列模型 / 图模型 / 多模态）→ 建 ANN → 拿 trigger item 找近邻。
- 优点：泛化好、能召回没共现过的语义相似 item（缓解冷启）。
- 缺点：解释性弱于共现。

> 工业实践：**Swing(共现) + 向量 i2i(泛化) 双路融合**，共现保准确、向量保覆盖。

---

## 三、线上链路

1. 取用户近期 trigger items（按时间/权重选 Top 几个）；
2. 每个 trigger 查 i2i 倒排表（共现）或 ANN（向量）取 TopN；
3. 多 trigger 结果加权合并（trigger 越近/权重越高贡献越大）；
4. 去重 + 过滤(已交互/违规/库存) + 截断。

---

## 四、版本迭代

### v0 · ItemCF
共现矩阵 + 余弦。问题：热门霸屏、活跃用户污染。

### v1 · Swing + 活跃度惩罚
换 Swing，加用户活跃度降权。效果：相似质量大幅提升，热门压制。

### v2 · 时间衰减 + trigger 加权
共现加时间衰减（近期共现权重高）；trigger 按近因/行为类型(购买>点击)加权。效果：时效性、相关性提升。

### v3 · 向量 i2i 融合
加一路 item embedding ANN，和 Swing 融合。效果：长尾/冷启 item 覆盖提升。

## 五、常见坑

**坑 1 · 热门 item 霸榜**
原因：ItemCF 下热门和谁都共现。解：换 Swing、活跃度惩罚、热度降权、召回后打散。

**坑 2 · 活跃用户污染相似度**
原因：什么都点的用户制造大量假共现。解：`1/log(1+|I_u|)` 降权，或截断超长用户序列。

**坑 3 · 共现稀疏 / 新 item 无共现**
原因：长尾、新品没有交互。解：向量 i2i 兜底（语义相似），或内容/多模态 embedding。

**坑 4 · 相似但不互补，推荐重复**
现象：买了手机一直推手机。解：类目过滤/配额、互补关系挖掘(buy-also-buy)、和重排多样性配合。

**坑 5 · 倒排表过大、更新慢**
解：每个 item 只存 TopK 相似；离线批量更新 + 热点增量。

## 六、常见问答（带追问）

**Q1：ItemCF 和 Swing 区别？为什么 Swing 更好？**
ItemCF 用"item 被多少共同用户交互"算相似，热门 item 和谁都像。Swing 用"用户对"度量，并惩罚共同交互多的活跃用户对，信号更干净、抗热门抗噪。
> 追问·Swing 里为什么除以 `|I_u ∩ I_v|`？ → 两个用户共同交互的 item 越少越"专一"，他们同时选 i、j 越能说明 i、j 真相似；活跃用户共同交互多，权重被压低。

**Q2：I2I 和 U2I 各自优劣？**
I2I：解释性强、新用户友好(1 个行为即触发)、稳；但偏"相似"易重复，依赖 trigger。U2I：整体刻画用户、多样性可控；但需要足够行为、解释性弱。工业上两者并存互补。

**Q3：共现类和向量类怎么选/融合？**
共现准但稀疏(长尾/新品弱)；向量泛化好但解释弱。融合：共现保头部准确、向量保长尾覆盖，分数归一后合并。

**Q4：trigger 怎么选？**
近期、高权重(购买>加购>点击)、去掉过期和噪声行为，通常取 Top 几个并按近因衰减加权。

**Q5：怎么从"相似"做到"互补"？**
Swing 给相似；互补要挖 buy-also-buy / 跨类目共现（手机→壳膜），或用序列的"下一个"信号。

---

## 七、选型一句话

- 要**稳、可解释、新用户能用** → I2I（Swing 主力）。
- 要长尾/冷启覆盖 → 加向量 i2i / 多模态 i2i。
- 要整体个性化、多样性 → U2I 双塔/MIND 配合。

---

## 八、监控指标

- 离线：Recall@K、相似 TopK 人工抽检准确率。
- 在线：召回覆盖率、热门集中度、重复率、类目多样性、pvctr/uvctr。
